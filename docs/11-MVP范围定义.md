# 11 —— MVP 范围定义

> Version 0.1 最小可玩版本。验证核心手感，再扩展。

---

## MVP 包含

### 地图（5 张）

| # | 地图 | 内容 |
|---|------|------|
| ① | 玩家的家 | 院子（烹饪灶台 + 未来设施预留位）+ 屋内（床/存档点 + 储物箱） |
| ② | 新手村 | 3~4 个 NPC（村长/引导人、杂货商、钓鱼向导、邻居），家入口 |
| ③ | 野外 A·晴原小径 | 2 种小怪、钓鱼点（河流）、钓鱼体验 NPC |
| ④ | 野外 B·林暗边缘 | 1 种小怪 + 1 种精英怪、第一章 Boss 出没点 |
| ⑤ | 小城镇 | NPC 商店、NPC 铁匠（修装备）、转职导师、烹饪 NPC |

### 功能系统

| 内容 | 范围 |
|------|------|
| 武器 | 剑 1 把（剑士） |
| 怪物 | 3~4 种（2 小怪 + 1 精英 + 1 Boss） |
| 宠物 | 宠物蛋获取 ×3 种（孵化后战斗） |
| 副职业 | **钓鱼 + 烹饪**（顺序解锁，都需体验任务后正式学） |
| 等级 | 1~15 级，Lv15 锁经验需转职 |
| 转职 | 剑士（唯一职业） |
| NPC 铁匠 | 修复装备耐久（修复价较高，为未来锻造师副职业铺垫） |
| 任务 | 主线第一章（12 个节点，约 100~110 分钟） |
| 存档 | 自动存档（进新地图）+ 手动存档（床/菜单） |
| 死亡 | 送回当前地图安全点，无惩罚 |

### 技术规格

| 项 | 值 |
|----|----|
| 分辨率 | 1280×720 最小窗口，**支持全屏自适应** |
| 帧率 | 60fps 渲染 |
| 像素密度 | 32×32（Boss 64×64） |
| 缩放策略 | 整数倍缩放，全屏时保持像素清晰 |

---

## MVP 不包含

| 内容 | 原因 |
|------|------|
| 后端/数据库/服务器 | 单机版用本地存储 |
| 交易市场频道系统 | 联机功能 |
| 世界聊天 / PvP / 组队 | 联机功能 |
| 副本/秘境/日常任务 | 主线结束后内容 |
| 法师/弓手职业 | MVP 只做剑士 |
| 炼金/锻造/种植/药剂副职业 | MVP 只做钓鱼+烹饪 |
| 锻造师副职业 | 用 NPC 铁匠替代 |
| 技能书系统 | V1 技能来源只走武器+护甲+职业天赋 |
| 季节/天气 | 预留字段，不实现 |
| 宠物进化 | 只有等级系统 |
| 角色立绘 | 纯对话框 |
| 太阳印记机制 | 仅剧情视觉，无玩法效果 |
| 第二章及后续剧情 | MVP 只做第一章 |

## MVP 验证目标

```
角色移动手感是否流畅？ → 走路 + 方向响应
全屏/窗口切换是否正常？ → 分辨率自适应
战斗是否"看得懂"？ → 怪物前摇 + 攻击判定 + 受击反馈
宠物 AI 是否不碍事？ → 跟随 + 攻击 + 防卡死
钓鱼是否有趣？ → 钓鱼条 + 鱼竿品质 + 鱼饵
烹饪是否有意义？ → 食谱 + 材料 + 效果应用
地图过渡是否顺滑？ → 渐黑渐亮 + 地图名飞入
存档是否安全？ → 自动不覆盖手动 + 战斗中禁存
Boss 战是否有戏剧感？ → 3 阶段 + 闪红光 + 伤害飘字
剧情节奏是否合理？ → 100~110 分钟完成第一章
```

---

## 开发路线

```
第 1 步（现在）：纯 MVP → 验证核心手感（2~4 周）
第 2 步（验证后）：缩小版全内容 → 接入所有系统极简版（4~8 周）
第 3 步（全程并行）：边做边讨论下一个系统
```
