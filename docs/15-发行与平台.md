## 为什么 Steam 优先

| 原因 | 说明 |
|------|------|
| 个人即可上架 | 不需要公司主体、不需要版号 |
| 押金可退 | $100 Steamworks 注册费，满 $1000 销售额后退还 |
| PC 操作适配零成本 | 你的游戏本来就是键鼠操作 |
| 独立游戏生态好 | Steam 对独立游戏友好，有发现机制 |
| 30% 抽成 | 行业标准，可接受 |

## Steam 上架流程（概念）

```
1. 注册 Steamworks 账号（https://partner.steamgames.com）
2. 签署 Steamworks 发行协议
3. 缴纳 $100 押金
4. 创建商店页面（截图、描述、标签、预告片）
5. 上传游戏构建版本
6. Steam 审核（1~2 周）
7. 发布
```

### 需要准备的东西

| 项 | 说明 |
|----|----|
| 游戏可玩版本 | 至少 MVP 完整版 |
| 商店截图 | 5 张以上 |
| 预告片 | 30~60 秒 |
| 游戏描述 | 中英文 |
| 标签 | RPG、像素、单人、冒险等 |
| 年龄分级问卷 | Steam 自动生成 |

## 版号知识（中国上架必知）

| 项 | 说明 |
|----|----|
| 什么是版号 | 游戏出版物号（ISBN），中国商业化必须 |
| 谁能申请 | 必须有公司主体（个人不行） |
| 流程 | 找出版单位合作 → 备案审查 → 6~12 个月 |
| 内容限制 | 暴力/赌博/魔改历史/敏感元素会被拒 |
| 对你的影响 | "太阳之下"是玄幻题材，可能触发内容审查，但轻度玄幻一般能过 |
| 时间 | 游戏完成后再考虑，现在不涉及 |

## 盈利模式

| 模式 | 说明 | 阶段 |
|------|------|:---:|
| **买断制** | Steam 上一次性购买 | MVP 首选 |
| 免费基础 + 外观付费 | 旧版 GDD 提到的模式 | 联机版考虑 |
| 免费体验 + 付费章节 | TapTap 路线 | 移动端 |

### MVP/Steam 阶段推荐：买断制

- 简单，个人能操作
- 不需要内购系统、商城系统
- 不需要后端验证购买
- 联机版再考虑免费+外观付费（需要 Nakama + 支付接入）

## 运维概念

运维 = 游戏上线后的持续维护。分阶段：

### MVP 阶段（单机 Steam）

| 运维项 | 说明 |
|--------|------|
| Bug 修复 | 玩家反馈 → 修代码 → 发补丁 |
| 版本更新 | Steam 管理后台推送新版本 |
| 存档兼容 | 更新时不能让玩家旧存档报废 |
| 社区维护 | Steam 社区/Discord 看反馈 |

### 联机阶段（未来）

| 运维项 | 说明 |
|--------|------|
| 服务器管理 | Nakama + PostgreSQL + Redis 的部署和监控 |
| 数据备份 | 定期备份数据库 |
| 玩家账号 | 账号系统维护、封禁、客服 |
| 反作弊 | 服务器权威验证 + 异常检测 |
| 支付运维 | 充值/退款/对账 |
| 经济调控 | 监控通胀，调整产出和消耗 |

## 中国个人独立开发者的现实路径

```
第 1 步（现在~3 个月）：MVP 完成
  → 在 Steam 上架免费测试版（或内部测试）
  → 不需要公司、不需要版号

第 2 步（3~6 个月）：正式版上架 Steam
  → 注册 Steamworks 个人账号
  → $100 押金
  → 买断制定价（建议 ¥30~60）
  → 不需要版号

第 3 步（6 个月后，视销量决定）：
  → 如果要国内商业化（WeGame/手机商店）
  → 注册个体工商户或公司
  → 申请版号
  → 这个阶段才需要真正考虑运维

第 4 步（联机版启动时）：
  → 部署 Nakama 服务器
  → 开始服务器运维
  → 考虑免费 + 外观付费
```

## 平台适配技术注意事项（现在就要预留）

| 项 | 现在怎么做 | 将来出移动端怎么做 |
|----|---------|------------------|
| 输入方式 | 键鼠事件 | 加触屏虚拟摇杆层 |
| 分辨率 | PC 1080p | 移动端自适应 |
| 存档 | 本地 JSON/SQLite | 接入云存档 |
| UI | 鼠标点击优化 | 触屏优化 |
| 资源 | 高清像素 | 压缩版本 |

Godot 4 支持同一项目导出多平台，但 UI 和输入需要分别适配。**现在不管，但代码结构上输入处理要和游戏逻辑分离**（和地基规矩 1 一致）。
