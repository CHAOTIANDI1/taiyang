## 帧率策略（双层独立）

| 层 | 帧率 | 说明 |
|----|------|------|
| 游戏渲染 | **60fps** | 画面刷新，追求流畅 |
| 动画播放 | **8~12fps** | 图切换速度，省美术工作量 |

两层独立。玩家看 60 帧丝滑画面，角色"抖"频率低——星露谷就这么做的。

---

## 极简帧方案

不做完整动画，用最少帧"假装在动"：

| 动作 | 帧数 | 帧数替代方案 |
|------|:---:|------|
| 行走 | 2 帧（左右脚交替） | 移动速度 + 脚步音效补偿 |
| 攻击 | 1 帧（挥剑 + 剑光特效） | 屏震 + 闪白 + 伤害数字补偿 |
| 受击 | 1 帧（闪白） | 着色器切换，0.1 秒回复 |
| 死亡 | 1 帧（变灰 + Alpha 淡出） | 渐变 + 粒子消散 |

## 美术工作流

```
第 1 步：AI 生概念图
  Gemini / SD / Midjourney → 生成角色站立图
  提示词模板：
  "pixel art character, 32x32, top-down view, idle pose,
   style: stardew valley + pokemmo, warm palette"

第 2 步：像素化
  导入 Aseprite → Sprite Size → 缩小到 32×32
  手动调整像素，去掉噪点

第 3 步：锁调色板
  固定 16~32 色调色板
  所有素材只在调色板内取色
  保证全局风格统一

第 4 步：做极简帧
  复制站立图 → 移动腿 1~2 像素 = 行走第 2 帧
  攻击不用动画，用剑光特效 + 闪白代码补偿

第 5 步：导出 Sprite Sheet
  Aseprite → Export → Sprite Sheet（长条图）
  Godot 导入后切割使用
```

## 打击感代码补偿方案（代替复杂动画）

当角色/怪物动作帧不够时，用代码补偿"看起来很爽"：

| 效果 | 代码实现 | 持续时间 |
|------|---------|---------|
| 命中闪白 | 着色器切换白色 → 回复 | 0.1 秒 |
| 屏幕震动 | 摄像机抖动 | 0.1 秒 |
| 命中定格 | 全游戏暂停（hitstop） | 0.05 秒 |
| 击退 | 怪物向后位移 | 0.2 秒 |
| 伤害飘字 | 数字上飘 + 淡出 | 0.8 秒 |
| 粒子特效 | 命中点冒粒子 | 0.3 秒 |

这些我全部能用 GDScript 写，不依赖美术素材。
