# 02 —— 战斗系统

---

## 操作方式

- 上下左右方向移动，**无翻滚、无连招**
- 不同武器拥有独立普通攻击模组
- **3 个主动技能键**，每个技能有独立的技能攻击模组
- 技能来源：
  - 武器自带技能
  - 护甲附带技能
  - 主职业天赋技能
  - **技能书**（学习指定技能，受职业和属性门槛限制）：**V1 不实现，后续版本启用**
- V1 技能来源只走前 3 项（武器 + 护甲 + 职业天赋）
- 技能书获取途径待后续版本规划

---

## 技能循环设计

| 键位 | 类型 | 冷却 | 作用 |
|------|------|------|------|
| 技能 1 | 基础技能 | 1~3 秒 | 短 CD 频繁使用，填充平砍空白 |
| 技能 2 | 核心技能 | 8~15 秒 | 伤害/效果强力 |
| 技能 3 | 爆发技能 | 8~15 秒 | 伤害/效果强力 |

## 攻击前摇与硬直

- 每种怪物攻击前必须有**肉眼可见的预警动画**（如红尾蝎翘尾巴 0.5 秒再刺）
- 玩家武器攻击：前摇 → 伤害帧 → 后摇，后摇期间不能立即取消
- 攻击判定用 Hitbox，近战怪判定框比视觉稍大一圈，防止贴脸穿模

## 攻击判定模式（Phase 4.4.6 步骤 2 落地）

### 设计目标

让怪物攻击既有"必中"也有"可躲"——前摇预警结束后玩家跑开就能 miss，不再无脑挨打。同时为步骤 3 DamageArea 抽象类 + 步骤 4 多形状扩展做铺垫。

### attack_pattern 字段（数据驱动）

数据来源：`data/monsters.json` 和 `data/pets.json` 的 `attack_pattern` 字段。

| 取值 | 含义 | 适用 |
|------|------|------|
| `guaranteed_hit` | 必中（前摇结束无条件扣血）| Boss 大招、不可躲技能 |
| `telegraph_then_check` | 前摇预警结束 → 再次检查目标是否在 attack_range 内 → 在则命中 / 不在则 miss | 红尾蝎（玩家跑开就 miss）|
| `instant_check` | 无预警，攻击动画期间用 AttackArea 持续检测命中 | 宠物（没有预警机制）|

### attack_shape 字段（步骤 3 DamageArea 落地用）

| 取值 | 形状 | 关键参数 |
|------|------|---------|
| `cone` | 扇形（朝向 + 半径 + 角度）| `attack_radius=68`、`attack_arc_degree=45` |
| `circle` | 圆形（全方向）| `attack_radius` |
| `rectangle` | 矩形（以自身为中心）| `width`、`height` |
| `rect_dir` | 朝向矩形（从自身向朝向延伸）| `length`、`width` |
| `cross` | 双十字矩形 | `length`、`width` |

### 数值推导（设计真值）

- **像素密度**：32×32（角色/怪物/NPC 统一，AGENTS.md §0 项目身份）
- **扇形半径**：32×√2×1.5 ≈ 68 像素（自身方块对角线 × 1.5）
- **扇形弧度**：45 度（自身中心点朝向 45 度弧）
- **推导链**：32（身位）→ 32×√2≈45.25（对角线）→ ×1.5≈67.88≈68（攻击半径）

### hit_count 字段（命中数量）

| 取值 | 含义 | 适用 |
|------|------|------|
| `1` | 单体最近（范围内只打最近的 1 个）| 玩家/宠物/红尾蝎 |
| `-1` | 群体（范围内全部命中）| 群体技能/AOE |
| `N`（N>1）| 范围内最近 N 个 | 链式闪电等 |

### 当前 MVP 配置

| 单位 | attack_pattern | attack_shape | attack_radius | attack_arc_degree | hit_count |
|------|---------------|--------------|---------------|-------------------|-----------|
| 玩家 | instant_check（J 攻击，无预警）| cone（步骤 3 落地）| 68 | 45 | 1 |
| 宠物（3 种）| instant_check | cone | 68 | 45 | 1 |
| 红尾蝎 | telegraph_then_check | cone | 68 | 45 | 1 |
| 训练木桩 | 无（static AI）| - | - | - | - |
| Boss（未来）| guaranteed_hit | 待定 | 待定 | 待定 | 待定 |

### 行为对照

- **改前（4.4.6 步骤 1 之前）**：怪物前摇结束 → 必中（玩家跑开也挨打）
- **改后（步骤 2）**：红尾蝎前摇结束 → 检查距离 → 在 attack_range=40 内则命中 / 不在则显示 miss 飘字（玩家跑开就 miss）

## DamageArea 工具类（Phase 4.4.6 步骤 3 + 步骤 4 落地）

### 设计目标

把"攻击范围判定"抽象成纯工具类，调用方只传 `shape_config` 即可切换形状，不改代码。满足 §0.5 三问：MVP 够用（扇形 + 单体最近）、加系统不重构（新形状只改 `_is_in_shape` 一处）、升级地基不改（碰撞层/数据驱动/存档不动）。

### 文件位置

`scripts/combat/damage_area.gd`（`class_name DamageArea`，纯工具类，不继承 Node）

### 调用接口

```gdscript
# 调用方负责提供粗筛候选（AttackArea.get_overlapping_bodies() 或场景遍历）
# DamageArea 负责按形状精筛 + 按 hit_count 取数量
var hits: Array = DamageArea.filter_targets(
    origin,          # 攻击者世界坐标 Vector2
    facing,          # 攻击者朝向 Vector2（归一化，扇形/朝向矩形用）
    shape_config,    # 形状配置 Dictionary
    candidates       # 候选目标 Array[Node2D]
)
```

### 形状使用示例（shape_config 配置）

| 形状 | shape_config 示例 | 适用场景 |
|------|------------------|---------|
| 扇形 cone | `{"type":"cone","radius":68,"arc_degree":45,"hit_count":1}` | 玩家/宠物/红尾蝎普攻（默认）|
| 圆形 circle | `{"type":"circle","radius":80,"hit_count":-1}` | AOE 技能/爆炸/群体伤害 |
| 矩形 rectangle | `{"type":"rectangle","width":64,"height":32,"hit_count":1}` | 玩家正面格挡区/陷阱区 |
| 朝向矩形 rect_dir | `{"type":"rect_dir","length":120,"width":40,"hit_count":1}` | 矛刺/直线冲刺攻击 |
| 双十字 cross | `{"type":"cross","length":120,"width":24,"hit_count":-1}` | Boss 十字斩/神圣十字 |

### hit_count 命中模式

| 值 | 含义 | 返回 |
|----|------|------|
| `1` | 单体最近 | `[最近的目标]` 或 `[]` |
| `-1` | 群体 | `[所有在形状内的目标]` |
| `N`（N>1）| 范围内最近 N 个 | `[最近 N 个目标]`（不足 N 个返回全部）|

### 数据驱动扩展方式

加新形状只需 3 步（无需改调用方代码）：
1. 在 `data/monsters.json` 或 `data/pets.json` 改 `attack_shape` 字段为新形状名
2. 在 `DamageArea._parse_shape_type()` 加 match 分支
3. 在 `DamageArea._is_in_shape()` 加 match 分支实现判定逻辑

### 自测方法（步骤 4 测试验证）

`DamageArea.run_self_test()` 静态函数已写好，覆盖 5 种形状 + hit_count 边界用例（共 16+ 项断言）。

**调用方式**（任选其一）：
1. **临时验证**：在 `world.tscn` 根节点的 `_ready()` 末尾加一行 `print(DamageArea.run_self_test())`，F5 运行后看控制台输出 `[DamageArea 自测] 通过 N / 失败 0` 即通过
2. **远程调试**：在 Godot 编辑器的"远程调试"面板手动调用 `DamageArea.run_self_test()`

**预期输出**：
```
[DamageArea 自测] 通过 16 / 失败 0（共 16 项）
true
```

### 当前 MVP 调用方

| 调用方 | 文件 | 调用位置 | shape_config 来源 |
|--------|------|---------|------------------|
| 玩家 J 攻击 | `scripts/player/player.gd` `_check_hit()` | hitbox 粗筛后精筛 | 常量 `ATTACK_SHAPE_CONFIG`（cone/68/45/1）|
| 宠物攻击 | `scripts/pet/pet.gd` `_check_pet_hit()` | AttackArea 粗筛后精筛 | `pets.json` 字段（cone/68/45/1）|
| 怪物攻击 | `scripts/enemy/monster.gd` `_do_attack()` telegraph_then_check 分支 | 单目标判定 | `monsters.json` 字段（cone/68/45/1）|

## 怪物 AI 模板

| 模板 | 行为 | 适用怪物举例 |
|------|------|------------|
| 近战冲锋型 | 走向玩家 → 攻击 → 冷却 → 再攻击 | 红尾蝎、石魔像 |
| 远程射击型 | 保持距离 → 投射物 → 玩家靠近就跑 | 暗影法师、毒蘑菇 |
| 自爆突进型 | 快速冲脸 → 范围自爆 | 爆裂史莱姆 |
| 环绕骚扰型 | 绕玩家转圈 → 偶尔冲刺 | 风翼蝙蝠 |
| 原地炮台型 | 不动 → 定时发射弹幕 | 机关石柱、毒花 |
| Boss 模板 | 多阶段 + 多技能 + 血量触发 | 所有 Boss |

每个怪物只需填参数：移动速度、攻击距离、攻击冷却、伤害数值。工作流：程序写一套模板逻辑，策划填表生成怪物。

## 脱战机制

怪物打不到玩家（玩家在高台/怪物卡住）时，3 秒后失去仇恨回原位并回满血。

---

## 战斗失败

- 回到上一个存档点 / 主城，或自己家中
- Boss 战死亡应在 Boss 门外重生（不是直接回家）
- **无死亡惩罚**（不掉落、不扣经验、不扣金币）
- 每个地图标记自己的安全点坐标，死亡时传送到当前地图的安全点

---

## Boss 战设计

- 分阶段：
  - 阶段 1（100%~60% HP）：基础技能 A 和 B
  - 阶段 2（60%~30% HP）：增加技能 C，A/B 稍微加快
  - 阶段 3（30%~0% HP）：所有技能加速/范围变大，可能加终极技能 D
- 切换阶段时 Boss 短暂停顿（1~2 秒），闪红光/咆哮，给玩家信号
- Boss 战斗区域判定：离开区域 → Boss 回满血

## 玩家攻击命中判定（Phase 3 实现）

- 玩家身上挂 `Area2D`（攻击 hitbox），平时 `monitoring=false`
- 按 J 触发攻击时 `_hitbox_area.monitoring = true` 持续 `attack_duration`（0.2 秒）
- hitbox 大小：50×50 矩形
- hitbox 位置：跟随角色"面向方向"摆在前方 `_facing × _hitbox_offset`
- 每物理帧调 `get_overlapping_bodies()` 拿当前重叠物体
- 检测物体是否有 `take_damage` 方法 → 调用并飘出伤害数字
- 一次攻击只命中一个怪（`_attack_hit` 防 60 帧重复扣）

### 攻击命中后的打击感补偿

- 屏震：摄像机偏移随机方向，强度 4.5，Tween 回原位 0.12 秒
- 伤害飘字：Label 上飘 40 像素+淡出 0.6 秒+销毁
- 怪物自身：`modulate = Color(3,3,3,1)` 闪白 → 0.1 秒 → 原色（Tween）

## 怪物 AI（Phase 3 实现）

### 近战冲锋型 AI 模板

- 找目标（`_target`）
- 算到玩家的距离向量
- 距离 > attack_range：走向玩家
- 距离 ≤ attack_range 且 CD OK：启动**前摇预警**（0.5 秒）
- 前摇中身体变红（`modulate = Color(2, 0.5, 0.5)`）
- 前摇结束：执行攻击 + 重置 CD

### 怪物死亡淡出

- hp≤0 时 `start_dying()` 发 `died` 信号
- 0.5 秒内 `modulate.a` 从 1 → 0
- 完成后 `queue_free()` 销毁

### 训练木桩

- `ai_template: "static"` 不移动
- 用于新手教学和测试攻击
- HP=30（3 次普攻砍死）
