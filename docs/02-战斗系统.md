# 02 —— 战斗系统

---

## 操作方式

- 上下左右方向移动，**无翻滚、无连招**
- 不同武器拥有独立普通攻击模组
- **3 个主动技能键**，每个技能有独立的技能攻击模组
- 技能来源：
  - 武器自带技能
  - 护甲附带技能
  - 主职业天赋技能
  - **技能书**（学习指定技能，受职业和属性门槛限制）：**V1 不实现，后续版本启用**
- V1 技能来源只走前 3 项（武器 + 护甲 + 职业天赋）
- 技能书获取途径待后续版本规划

---

## 技能循环设计

| 键位 | 类型 | 冷却 | 作用 |
|------|------|------|------|
| 技能 1 | 基础技能 | 1~3 秒 | 短 CD 频繁使用，填充平砍空白 |
| 技能 2 | 核心技能 | 8~15 秒 | 伤害/效果强力 |
| 技能 3 | 爆发技能 | 8~15 秒 | 伤害/效果强力 |

---

## 攻击前摇与硬直

- 每种怪物攻击前必须有**肉眼可见的预警动画**（如红尾蝎翘尾巴 0.5 秒再刺）
- 玩家武器攻击：前摇 → 伤害帧 → 后摇，后摇期间不能立即取消
- 攻击判定用 Hitbox，近战怪判定框比视觉稍大一圈，防止贴脸穿模

## 怪物 AI 模板

| 模板 | 行为 | 适用怪物举例 |
|------|------|------------|
| 近战冲锋型 | 走向玩家 → 攻击 → 冷却 → 再攻击 | 红尾蝎、石魔像 |
| 远程射击型 | 保持距离 → 投射物 → 玩家靠近就跑 | 暗影法师、毒蘑菇 |
| 自爆突进型 | 快速冲脸 → 范围自爆 | 爆裂史莱姆 |
| 环绕骚扰型 | 绕玩家转圈 → 偶尔冲刺 | 风翼蝙蝠 |
| 原地炮台型 | 不动 → 定时发射弹幕 | 机关石柱、毒花 |
| Boss 模板 | 多阶段 + 多技能 + 血量触发 | 所有 Boss |

每个怪物只需填参数：移动速度、攻击距离、攻击冷却、伤害数值。工作流：程序写一套模板逻辑，策划填表生成怪物。

## 脱战机制

怪物打不到玩家（玩家在高台/怪物卡住）时，3 秒后失去仇恨回原位并回满血。

---

## 战斗失败

- 回到上一个存档点 / 主城，或自己家中
- Boss 战死亡应在 Boss 门外重生（不是直接回家）
- **无死亡惩罚**（不掉落、不扣经验、不扣金币）
- 每个地图标记自己的安全点坐标，死亡时传送到当前地图的安全点

---

## Boss 战设计

- 分阶段：
  - 阶段 1（100%~60% HP）：基础技能 A 和 B
  - 阶段 2（60%~30% HP）：增加技能 C，A/B 稍微加快
  - 阶段 3（30%~0% HP）：所有技能加速/范围变大，可能加终极技能 D
- 切换阶段时 Boss 短暂停顿（1~2 秒），闪红光/咆哮，给玩家信号
- Boss 战斗区域判定：离开区域 → Boss 回满血

## 食物战斗使用

战斗中吃食物有**冷却**（15 秒一次），且有 1~2 秒吃东西动画，期间不能攻击/移动。

## 玩家攻击命中判定（Phase 3 实现）

- 玩家身上挂 `Area2D`（攻击 hitbox），平时 `monitoring=false`
- 按 J 触发攻击时 `_hitbox_area.monitoring = true` 持续 `attack_duration`（0.2 秒）
- hitbox 大小：50×50 矩形
- hitbox 位置：跟随角色"面向方向"摆在前方 `_facing × _hitbox_offset`
- 每物理帧调 `get_overlapping_bodies()` 拿当前重叠物体
- 检测物体是否有 `take_damage` 方法 → 调用并飘出伤害数字
- 一次攻击只命中一个怪（`_attack_hit` 防 60 帧重复扣）

### 攻击命中后的打击感补偿

- 屏震：摄像机偏移随机方向，强度 4.5，Tween 回原位 0.12 秒
- 伤害飘字：Label 上飘 40 像素+淡出 0.6 秒+销毁
- 怪物自身：`modulate = Color(3,3,3,1)` 闪白 → 0.1 秒 → 原色（Tween）

## 怪物 AI（Phase 3 实现）

### 近战冲锋型 AI 模板

- 找目标（`_target`）
- 算到玩家的距离向量
- 距离 > attack_range：走向玩家
- 距离 ≤ attack_range 且 CD OK：启动**前摇预警**（0.5 秒）
- 前摇中身体变红（`modulate = Color(2, 0.5, 0.5)`）
- 前摇结束：执行攻击 + 重置 CD

### 怪物死亡淡出

- hp≤0 时 `start_dying()` 发 `died` 信号
- 0.5 秒内 `modulate.a` 从 1 → 0
- 完成后 `queue_free()` 销毁

### 训练木桩

- `ai_template: "static"` 不移动
- 用于新手教学和测试攻击
- HP=30（3 次普攻砍死）

## 过场系统架构（IntroSequence 控制器）

- 一个控制过场内容的脚本，负责按顺序播放分镜/文字/动画
- MVP 实现：版本 A（黑屏 + 文字独白）
- 接口预留：可替换为版本 B 漫画分镜 / 版本 C 动画
- 适用场景：开场独白、Boss 一生展示、章节结语动画
- 详见 `docs/12-技术栈与架构.md` 过场系统架构
