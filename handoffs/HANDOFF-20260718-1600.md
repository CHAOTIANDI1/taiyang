## 9. 工具切换注意事项（OpenCode → TRAE）

| 维度 | OpenCode 版 sliver-vibe-coding | TRAE 内置版 | 结论 |

| references 路由 | routes-index.md | 完全一致 | ✅ |

**结论**：TRAE 内置版就是 Gitee 源版的 runtime bundle，无需额外转换。

### 9.2 Git 推送差异

| 维度 | OpenCode | TRAE |
|------|---------|------|
| Git 操作 | 通过 WSL 终端执行 | 通过 RunCommand 在 Windows PowerShell 执行 |
| 提交格式 | `[类型] 内容，已同步 XX.md` | 相同（遵循 AGENTS.md §1.6） |
| Push | `git push` in WSL | `git push` in PowerShell |
| 换行符 | .gitattributes 统一 | 相同（已配置 *.gd eol=lf） |

**注意事项**：
- TRAE 的 RunCommand 在 PowerShell 执行，路径用反斜杠 `\`
- Git 操作直接在 `D:\taiyang` 目录执行即可
- 中文文件名（知识库/）在 PowerShell 里需要引号包裹

### 9.3 模型一致性

- OpenCode 用的模型：`opencode-go/glm-5.2`
- TRAE 用的模型：`GLM-5.2`（用户已确认切换）
- **同一个模型，能力一致**
