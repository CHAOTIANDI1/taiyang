# 10 - 上下文管理与换对话

## 概念

**上下文** = AI 脑子里记着的对话内容。容量有限，满了最早的内容被挤出去——AI 就"忘事"。

换对话 = 开新窗口或恢复旧窗口。**怎么不丢进度**是关键。

## 功能

- 让你能判断什么时候该换对话
- 换对话时不丢项目状态
- 让新会话 AI 也能像老 AI 一样工作

## 运作方式

### 信号识别（什么时候该换）

| 信号                 | 含义            | 行动  |
| ------------------ | ------------- | --- |
| AI 让你重提之前确认过的事     | 早期对话被挤出去了     | 准备换 |
| AI 忘了你之前选的方案 A/B/C | 同上            | 准备换 |
| 系统提示"上下文已满"        | 接近上限          | 必须换 |
| AI 给的代码开始不符合规矩     | 它忘了 AGENTS.md | 必须换 |
| 隔了 1 周再回来          | 太早内容都已模糊      | 建议换 |
|                    |               |     |

### 换对话的两种方式

**方式 1: 继续同一对话（恢复历史）**
```powershell
opencode -c
```
或：
```powershell
opencode -s ses_0969d65baffevD7MA8n8Ba0vkl
```
恢复历史，AI 还记得前面说的。**容量还是那么多**——满了恢复回来也还是满的。

**方式 2: 开全新对话（推荐当原对话满了）**
```powershell
cd D:\taiyang
opencode
```
新会话**第一句话必须**：
> 请先读 `D:\taiyang\AGENTS.md` 和 `D:\taiyang\docs\00-总览.md`，用 5 句话复述项目状态后告诉我，然后等我下一步指示。

这是"对话开始协议"（防塌方方案 1）。

### 关键原理

新对话的 AI 是**完全空白的**，但只要它读了 AGENTS.md 和 docs/00，认知就能赶上老 AI 的 80%~90%。因为**我们把所有真值写进了文档**。

生活类比：不让会计记所有账在脑子里，写进账本。换会计了，新人看账本就能接着干。

## 原理

```
传统 AI 协作：AI 脑子里记 → 脑子满了忘事 → Bug 增多 → 项目失控
我们的方式：所有真值写进 .md → AI 每次都先读文档 → AI 不依赖记忆
```

文档驱动 = **抗遗忘的工程化措施**。

### 关键加强：上下文预算协议（v2 新增）

每次完成一段实质性工作，AI 自检上下文使用量：

```
估算当前会话上下文已用：
- < 30%：安心继续
- 30%~60%：正常，但完成下个任务后建议换对话
- 60%~80%：建议立刻写完会话交接文档，准备换对话
- > 80%：必须立刻写交接并换对话
```

到 60% 以上时 AI 必须主动提醒："上下文已用 65%，下次再聊该换新对话了。**我准备写一份会话交接文档**。"

## 优势

| 优势 | 说明 |
|------|------|
| 防止 AI 忘事 | 文档是脑子的外部记忆 |
| 换 AI/换模型无感 | 新 AI 5 分钟进入状态 |
| 一年项目不塌方 | 跨越 N 次会话依然一致 |
| 不被容量绑架 | 满就换，文档兜底 |

## 使用场景

### 场景 1：今天聊到一半 AI 开始重复问问题

你跟它说："上下文是不是满了？写一份会话交接文档给我。" → AI 停下来写交接 → 你保存 → 开新对话。

### 场景 2：换电脑/换 AI 工具

新电脑装好 OpenCode → 新会话 → 第一句让 AI 读 AGENTS.md → 5 句话确认 → 继续干活。

### 场景 3：隔了 3 天再回来

你打开 OpenCode → 写 `opencode -c` 恢复历史会话。如果旧会话上下文已满，AI 会主动提醒"该写交接文档了"。
