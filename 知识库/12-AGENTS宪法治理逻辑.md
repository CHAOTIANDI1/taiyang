# 12 - AGENTS 宪法治理逻辑

## 概念

**AGENTS.md = 项目宪法**，比 docs/ 里的设计文档地位高一等。规定了所有 AI 必须遵守的规矩。

生活类比：
- AGENTS.md = 国家宪法（最高法）
- docs/00-总览.md = 政府公告
- docs/01~15 = 各部门规章
- scripts/ = 执行的法律条文
- data/*.json = 罚款金额等具体数值

## 功能

让任何 AI 进项目时都能立刻知道：
- 这是什么项目、什么技术栈
- 有哪些铁律不能违反（地基三规矩）
- MVP 阶段不能碰什么（范围红线）
- 怎么改代码同步改文档
- 怎么加新东西不推翻地基

## 运作方式

### 7 条防塌方方案的环环相扣

```
方案 1: 对话开始协议     → 防 AI 不读规矩瞎写
方案 2: 三项一致原则     → 防改代码忘改文档
方案 3: 数据文件清单     → 防数字散落各处
方案 4: 能力总目录       → 防加新系统没地基
方案 5: 变更影响协议     → 防新技术推翻旧设计
方案 6: MVP 验收清单     → 防半成品过关
方案 7: 合规巡检         → 定期自查防违规堆积
```

### 整体逻辑流

```
新会话开始
  → 方案 1 对话开始协议（AGENTS.md → docs/00 → docs/12）
  → 复述理解
  → 用户确认
  → 开干任务
  → 完成 → commit 检查方案 2（三项一致）
  → 想加新东西 → 方案 5（4 问协议）
  → 新东西涉及 JSON → 方案 3 查清单
  → 新东西涉及系统 → 方案 4 查总目录
  → 里程碑完成 → 方案 6 验收清单全过
  → 准备关机/换会话 → 方案 7 合规巡检
  → 写 HANDOFF.md
```

## 原理

**"治理 = 规矩 + 执行 + 审计"** 三层：
- **规矩层**：AGENTS.md 写死所有规矩
- **执行层**：AI 干活时被规矩约束（如 commit 必须写同步标签）
- **审计层**：合规巡检定期检查

类似：公司有制度（员工手册）→ 执行（员工按手册工作）→ 审计（财务季度查账）。三层都有才不塌方。

## 优势

| 优势 | 说明 |
|------|------|
| 抗 AI 遗忘 | AI 不靠记忆靠文档 |
| 跨 AI 兼容 | OpenCode/Claude/GPT 都能读 .md |
| 防地基松动 | 加新东西先查清单 |
| 项目可追溯 | Git 历史里看每次提交 |
| 一年不塌 | 文档/数据/代码永远同步 |

## 使用场景

### 场景 1：新 AI 会话开始

第一句话："读 AGENTS.md 和 docs/00-总览.md，复述项目状态"。
AI 读完毕 → 用 5 句话告诉你它懂了什么 → 你确认"对，继续..."。

### 场景 2: 我（用户）临时想加一个新系统

不直接开干。让 AI 走方案 5（4 问协议）：
1. docs 里提过没？
2. 影响哪些现有系统？
3. 加哪个 JSON？
4. 三项一致吗？

AI 答 4 问答完 → 你确认 → 才动。

### 场景 3：AI 改完一段代码

提交格式强制：`[新增] XX 系统，已同步 XX.md 和 XX.json`。
不写"已同步"被视为未完成。下次会话开始时合规巡检会检查这个。

### 场景 4：用户发现 AI 没遵守规矩

直接告诉 AI "你违反 AGENTS.md §1.2 数据表驱动了，代码里出现了硬编码 `if name == '红尾蝎'`" → AI 自查修代码 → 补提交说明"修复 AGENTS §1.2 违规"。
