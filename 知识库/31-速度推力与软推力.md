# 31 - 速度、推力与软推力（Velocity / Push Force）

## 概念

**速度（Velocity）** = 物体每秒钟往哪个方向移动多远。在 Godot 里用 `Vector2(x, y)` 表示，x 是水平方向（正=右，负=左），y 是垂直方向（正=下，负=上）。

**推力（Push Force）** = 给一个物体一个速度，让它往特定方向移动一段距离。不是永久速度，是"推一把"后慢慢停下来。

**软推力（Soft Push Force）** = 不靠物理引擎的硬碰撞，靠代码给速度来"推开"物体。推力大小、方向、衰减都可以调。

生活比喻：
- **速度** = 你走路的速度。每秒走 1 米就是速度 1，每秒跑 5 米就是速度 5。
- **推力** = 有人推你一把。你不会被挡住（不是撞墙），但你会往被推的方向滑动几步，然后停下来。
- **软推力 vs 物理碰撞**：物理碰撞像两个人面对面撞上互相挡住；软推力像有人推你一把你滑开但不挡路。

**反例 vs 正例**：
```
❌ 错误（物理碰撞）：玩家和怪物相撞，互相挡住
   → 玩家被挤到墙角，所有方向卡死

✅ 正确（软推力）：怪物靠近时，代码给玩家一个速度让玩家被推开
   → 玩家和怪物不互相挡住（不卡墙）
   → 但玩家会被怪物"挤"开（有物理感）
   → 推力大小可调，方向可算，可加衰减
```

## 功能

- **速度**：控制物体每帧移动多少，是所有移动的基础
- **推力**：临时给物体一个速度，让它被"推"一下
- **软推力**：用 Area2D 检测靠近 + 代码给速度，替代物理碰撞，避免卡墙

## 运作方式

### 速度怎么用（本项目已有代码）

```gdscript
# player.gd 里的移动代码
var direction: Vector2 = Input.get_vector("move_left", "move_right", "move_up", "move_down")
# direction 是一个方向向量，比如向右走就是 Vector2(1, 0)

velocity = direction * speed
# speed=250 表示每秒移动 250 像素
# 向右走时 velocity = Vector2(250, 0)，每秒往右移 250 像素

move_and_slide()
# 这一行让 Godot 按 velocity 的值移动一帧（1/60 秒）
# 如果碰墙会自动沿墙滑动
```

### 推力怎么实现（方案 C 示例，未来用）

```gdscript
# 怪物身上的 Area2D 检测到玩家进入时
func _on_push_area_body_entered(body):
    if body.name == "Player":
        # 算出"从怪物指向玩家"的方向
        var push_direction = (body.global_position - global_position).normalized()
        # 给玩家一个推力速度（往远离怪物的方向）
        body.velocity = push_direction * 300  # 推力大小 300
        # 玩家会被推开，但不互相挡住
```

### 代码逐行执行流程（软推力方案 C）

```
第 1 步：怪物身上挂一个 Area2D（警戒区），半径比如 40 像素
第 2 步：玩家走进警戒区 → Area2D 发出 body_entered 信号
第 3 步：代码算出"从怪物指向玩家"的方向（normalized 让长度=1）
第 4 步：给玩家 velocity = 方向 × 推力大小
第 5 步：玩家的 _physics_process 里 move_and_slide() 按这个速度移动
第 6 步：推力衰减（每帧 velocity 乘 0.9，慢慢减速停下来）
```

### 衰减是什么（让推力自然停下来）

```gdscript
# 玩家被推后，每帧让速度慢慢减小
func _physics_process(delta):
    velocity *= 0.9  # 每帧速度变成原来的 90%
    # 10 帧后速度变成 0.9^10 ≈ 0.35（原来的 35%）
    # 30 帧后速度变成 0.9^30 ≈ 0.04（几乎停下）
    move_and_slide()
```

生活比喻：你被推一把后不会一直滑，摩擦力让你慢慢停下来。代码里用"每帧乘 0.9"模拟这个摩擦力。

## 原理

Godot 的移动物体有两种：
- **CharacterBody2D**：手动控制速度，`move_and_slide()` 按速度移动，碰墙会沿墙滑动
- **RigidBody2D**：物理引擎自动模拟，有质量、惯性、摩擦力（本项目不用，太复杂）

**软推力的核心思路**：用 CharacterBody2D（手动控制速度），不靠物理碰撞（不用 RigidBody2D），靠代码在 Area2D 检测到靠近时给一个速度。这样：
- 物理碰撞层可以设成互不碰撞（不卡墙）
- 但代码层面给速度让玩家被"推"开（有物理感）

**为什么不用 RigidBody2D**：RigidBody2D 的物理模拟太复杂，调参难，容易出 bug。CharacterBody2D 手动控制速度更可控，符合"先跑通再优化"原则。

## 优势

| 优势 | 说明 |
|------|------|
| 不卡墙 | 不靠物理碰撞，永不卡在墙角 |
| 有物理感 | 玩家会被怪物推开，比方案 A（互不碰撞）更有打击感 |
| 推力可调 | 大小（轻轻推/用力推）、方向（从怪物指向玩家）、衰减（快速停/慢速停）都能调 |
| 可复用 | 击退、爆炸推力、宠物避让都能用同一套逻辑 |
| 可控性强 | 比 RigidBody2D 物理模拟更可控，bug 少 |

## 使用场景

### 本项目的升级路径

- **MVP 阶段（当前）**：方案 A（碰撞层分离，互不碰撞），零代码零风险
- **联机版升级目标**：方案 C（Area2D + 软推力），怪物推玩家有打击感
- **升级时不改地基**：碰撞层规范（Layer 1=墙/2=玩家/4=怪物）不动，只在怪物脚本加 Area2D + 推力代码

### 软推力可复用的场景

- **怪物推玩家**：怪物靠近时把玩家推开
- **击退效果**：玩家攻击怪物时，怪物被推退一段距离
- **爆炸推力**：炸弹爆炸时把周围的怪和玩家推开
- **宠物避让**：宠物跟随玩家时，互相靠近会被软推开避免重叠
- **流水推力**：水流区域把进入的物体往下游推

### 架构地基三问（方案 C）

| 问 | 回答 |
|----|------|
| ① MVP 够用吗？ | ✅ 但需额外写 Area2D 推力代码 + 调参，MVP 不必 |
| ② 加系统重构吗？ | ✅ 可抽象为 `SoftForceArea` 组件，被击退/推挤/宠物避让复用 |
| ③ 升级地基改吗？ | ✅ 本身是升级终点，地基（碰撞层规范）和 A 相同 |

## 反例 vs 正例（常见错误）

### 错误 1：推力不衰减，玩家一直滑
- **原因**：给了 velocity 后没有每帧衰减，玩家会一直往被推的方向滑
- **修复**：每帧 `velocity *= 0.9`（或用 Tween 做平滑减速）

### 错误 2：推力方向算反了
- **原因**：`body.global_position - global_position` 算反了，变成把玩家拉向怪物
- **修复**：记住"从怪物指向玩家"= `玩家位置 - 怪物位置`

### 错误 3：推力太大把玩家推出墙
- **原因**：推力速度太大，一帧移动距离超过墙的厚度
- **修复**：推力大小适中（300 左右），且玩家 mask=1 仍碰墙，move_and_slide 会沿墙滑动
