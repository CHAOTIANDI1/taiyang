## 概念

**GDScript 是强类型语言**：变量一旦用 `var name: Type` 显式声明类型，**赋值的类型必须与声明类型兼容**，否则编译器在解析阶段（还没运行游戏）就报错。

**生活比喻 1（生动形象）**：变量声明类型像"容器形状"
- `var empty_hits: Array` = 我要一个**糖罐**（装糖的容器）
- `_get_nearest()` 返回 `Node` = 函数给我**一颗糖**
- 把一颗糖塞进糖罐形状的容器——形状对不上，编译器立刻喊停："糖不是糖罐！"

**生活比喻 2（本游戏应用）**：4.4.6 我写 DamageArea 自测函数时
- `_get_nearest()` 函数签名是 `-> Node:`，明确返回"一颗节点（Node）"或"空手（null）"
- 我声明 `var empty_hits: Array` 是说"我要一个**数组**"
- 一颗节点塞不进数组形状的变量——类型不兼容，编译报错
- 修复就是"把容器换成糖的形状"：`var empty_hits: Node`

**反例 vs 正例**：
```gdscript
❌ 错误（声明 Array，赋值 Node）：
var empty_hits: Array = _get_nearest(origin, [])  # Node 不能塞进 Array 容器

✅ 正确（声明 Node，赋值 Node）：
var empty_hits: Node = _get_nearest(origin, [])   # 类型一致
```

## 运作方式

### 报错原文与翻译

```
Parser Error: Cannot assign a value of type Node to variable "empty_hits" with specified type Array.
```

**翻译**：无法把 `Node` 类型的值赋给声明为 `Array` 类型的变量 `empty_hits`。

**关键字拆解**：
- `Parser Error` = 解析阶段错误（编译期，还没运行游戏）
- `Cannot assign` = 无法赋值
- `a value of type Node` = 右边的值是 Node 类型（来自 `_get_nearest()` 返回值）
- `to variable "empty_hits"` = 赋值给变量 empty_hits
- `with specified type Array` = 因为这个变量被显式声明为 Array 类型

### 出错代码

```gdscript
# damage_area.gd 第 341 行（修复前）
var empty_hits: Array = _get_nearest(origin, [])
```

### 函数签名核对

```gdscript
# damage_area.gd 第 159 行
static func _get_nearest(origin: Vector2, candidates: Array) -> Node:
    # ↑ 返回类型是 Node（最近节点 or null）
    if candidates.is_empty():
        return null
    ...
```

**对照**：声明 `Array` vs 返回 `Node` → 不兼容 → 编译报错。

### 修复代码

```gdscript
# damage_area.gd 第 341 行（修复后）
var empty_hits: Node = _get_nearest(origin, [])
```

### 代码逐行执行流程

**场景**：自测函数验证"`_get_nearest` 收到空数组应返回 null"

```
1. var empty_hits: Node = _get_nearest(origin, [])
   - 编译期检查：
     a. 看 _get_nearest 返回类型 → Node
     b. 看 empty_hits 声明类型 → Node
     c. 两者一致 → 编译通过
   - 运行期执行：
     a. _get_nearest(origin, []) 被调用
     b. candidates.is_empty() = true → 返回 null
     c. empty_hits = null（合法，因为 null 是 Node 类型的合法空值）

2. if empty_hits == null:
   - null == null → true → pass_count += 1（自测通过）
```

## 原理

### 为什么 GDScript 4 要做静态类型检查？

GDScript 4 引入严格静态类型检查（4.7 进一步收紧），目的是：
- **把 Bug 提前到编译期**：原本要 F5 跑游戏才发现"塞错了类型"，现在保存文件就报错
- **性能更好**：引擎知道确切类型后可以优化字节码
- **IDE 补全更准**：编辑器知道 `empty_hits` 是 `Node`，能自动提示 Node 的方法

代价是小白容易踩坑：声明类型与赋值类型不一致就立刻报错，不像 Python/JS 那样"运行时再说"。

### 为什么 `Node` 和 `Array` 不兼容？

GDScript 的类型系统是**严格层级**：
- `Node` 是 Godot 节点基类（继承自 Object）
- `Array` 是容器类（不继承 Node）
- 两者在类型树上**没有任何继承关系**，互不兼容

只有以下情况类型兼容：
1. **完全相同**：`var x: Node = Node.new()`
2. **子类赋值给父类**：`var x: Node = CharacterBody2D.new()`（CharacterBody2D 是 Node 的子类）
3. **显式类型转换**：`var x: int = int("42")`（字符串转整数）

### 为什么 `null` 能赋值给 `Node`？

GDScript 把 `null` 视为**所有对象类型的合法空值**：
- `var x: Node = null` ✓（Node 可以是 null）
- `var x: Array = null` ✗（Array 不是对象类型，null 不能塞进 Array 容器——除非显式声明为 `Variant`）

所以 `_get_nearest()` 返回 `Node` 时，"找不到目标返回 null"是合法的。

### 编译期 vs 运行期

- **编译期（Parser Error）**：保存文件时编辑器/Godot 解析代码，发现类型不匹配立刻报错
- **运行期（Runtime Error）**：F5 跑游戏时才暴露的问题（如除以 0、空指针访问）

本笔记的报错属于**编译期**——游戏还没跑就喊停，好处是"早发现早修复"。

## 使用场景

### 场景 1：声明变量接函数返回值

**核对清单**：
1. 看函数签名 `-> XXX`
2. 变量声明类型与 `XXX` 一致（或 XXX 的父类）
3. 不能用无关类型

```gdscript
# ✅ 正确
static func _get_nearest(...) -> Node:
    ...
var result: Node = _get_nearest(...)   # 一致

# ✅ 正确（用父类接子类返回值，多态）
static func _make_pet() -> Pet:
    ...
var p: Node = _make_pet()              # Pet 是 Node 子类，合法

# ❌ 错误（类型不兼容）
var result: Array = _get_nearest(...)  # Node ≠ Array
```

### 场景 2：自测代码常踩的坑

写自测函数时容易"凭脑子里的上下文"写错类型——比如本次我因为"在测数组相关逻辑"，就顺手把 `empty_hits` 声明为 `Array`，但其实测的是 `_get_nearest` 返回值，应该是 `Node`。

**防错口诀**："声明类型看右边返回值，不看场景语境"

### 场景 3：函数返回 null 的处理

```gdscript
# 返回类型是 Node，可以返回 null
var x: Node = _get_nearest(origin, [])
if x == null:
    print("没找到目标")
else:
    print("找到了：", x.name)
```

注意：`null` 能赋给对象类型（Node/Node2D/CharacterBody2D 等），但不能赋给容器类型（Array/Dictionary）。

### 与 DamageArea 工具类对接

- DamageArea 是纯工具类，所有函数都是 `static func`
- 静态函数的返回类型在签名里明确声明，调用时变量声明必须匹配
- 自测函数 `run_self_test()` 覆盖 5 种形状 + hit_count 边界，每一行赋值都要类型匹配

### 与 22 号笔记互补

- 22 号讲"用 `:=` 推断不出来"——解决方法是**显式声明类型**
- 46 号讲"显式声明的类型与赋值类型不匹配"——解决方法是**对齐声明类型与函数返回类型**
- 两者都是 GDScript 4 静态类型系统的常见报错，但触发条件不同
