# 15 - Godot 碰撞与粒子体积

> 做战斗系统时才能具体讲清楚。先把概念讲明白，到时候真做时我同步补充具体操作步骤。

## 概念

- **碰撞**：物体 A 碰到物体 B，Godot 自动检测到，触发事件。比如玩家撞墙不动、玩家攻击范围扫到怪物就触发伤害。
- **粒子**：由许多小图形组成的特效系统。比如命中"喷出火星"、死亡"消散烟雾"。
- **体积**：碰撞的范围有多大（攻击 box 多大决定能不能砍到怪物）。

## 功能

- 碰撞 = "谁碰到了谁"的物理判定
- 粒子 = 纯视觉特效（不影响玩法，只造打击感）
- 体积 = 攻击/受伤判定的范围大小

## 运作方式（先理解概念）

### 碰撞体的类型

| Godot 节点 | 用途 |
|-----------|------|
| `CollisionShape2D` | 给物体加碰撞体（撞墙不穿）|
| `Area2D` | 一个区域，谁进来触发事件（攻击范围、陷阱）|
| `RayCast2D` | 一条射线，检测前方障碍（视野） |

### 碰撞层（Collision Layer/Mask）

Godot 可以分 32 个"碰撞层"。比如：
- Layer 1: 玩家
- Layer 2: 怪物
- Layer 3: 墙
- Layer 4: NPC

设置 mask 让谁跟谁碰：
- 玩家 mask = Layer 3（碰墙）+ Layer 4（碰 NPC）
- 怪物 mask = Layer 1（追玩家）+ Layer 3（碰墙）

**那么玩家不会攻击怪物的团块阻挡，但攻击 Area2D 触发伤害**。

### 攻击判定的体积反例/正例

反例：挥剑动画看起来砍到了怪，但 hitbox 太小 → 没触发伤害。

正例：在剑的动画关键帧加一个比剑身稍大的 `Area2D`，怪物进入这个 area → 触发伤害。

我们设计的"Hitbox 比视觉稍大一圈"就是 AGENTS.md §提到过的——防贴脸穿模。

### 粒子

Godot 的 `CPUParticles2D` 节点：
- 配置"喷出多少个粒子"/"粒子是什么颜色"/"向哪个方向喷"/"喷多久"
- 代码：
```gdscript
func on_hit():
    var particles = CPUParticles2D.new()
    particles.amount = 8    # 8 个粒子
    particles.lifetime = 0.3  # 0.3 秒消失
    # 添加到命中点
```

## 原理

- 碰撞：Godot 每帧 60 次检查所有碰撞体的重叠，事件回调你写的函数
- 粒子：纯渲染层逻辑，不影响游戏数据，只画给玩家看

## 优势

| 优势 | 说明 |
|------|------|
| 碰撞让世界"实在" | 不穿墙、有边界 |
| 攻击判定公平 | 玩家看得到的原则上能砍到 |
| 粒子放打击感 | 0.1 秒火星让命中"有重量" |

## 使用场景

### 场景 1：玩家挥剑攻击

1. 在剑的 `AnimatedSprite2D` 加一个 `Area2D` 子节点
2. Area2D 内放 `CollisionShape2D`（一个矩形或扇形，包住剑的范围）
3. 设置 Area2D 的 mask = Layer 2（怪物层）
4. 怪物进入 Area2D → 触发 `body_entered` 信号 → 调攻击逻辑
5. 同时启动粒子节点 → 喷火星 0.3 秒

### 场景 2：怪物死亡特效

不用动画。直接：
- 怪物_sprite 淡出（Alpha 0.5 秒）
- 同时撒粒子（"消散"效果）
- 0.6 秒后销毁节点

这就是 MVP §极简帧里"死亡 1 帧（变灰+淡出）+ 粒子消散"的落地。
