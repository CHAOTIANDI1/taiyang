# 05 - AGENTS.md 宪法治理逻辑

> 这个章节解释：为什么我们要把规矩写进 AGENTS.md，它和别的文档有什么不同。

---

## AGENTS.md 是什么

**生活类比**：项目宪法。比 docs/ 里的设计文档**地位高一等**。

| 文档种类 | 地位 | 类比 |
|----------|:---:|------|
| AGENTS.md | **最高** | 国家宪法 |
| docs/00-总览.md | 高 | 政府公告 |
| docs/01~15 | 中 | 各部门规章 |
| docs/14/15 美术/发行 | 中 | 部门规章 |
| 知识库/ | 辅助 | 个人学习笔记，对 AI 没强制力 |
| 代码 scripts/ | 执行 | 实际执行的法律条文 |
| 数据 data/ | 执行参数 |罚款金额等具体数 |

---

## 为什么 AGENTS.md 最高

因为它写了**所有 AI 都必须遵守的铁律**：
1. 项目身份（是什么游戏、什么技术栈、什么平台）
2. 地基三规矩（数据/逻辑/存档怎么写）
3. MVP 范围红线（什么不能做）
4. 文档管理规矩（先读再改、改代码同步改文档）
5. 代码结构和命名约定
6. 安全禁止事项
7. 决策权限（谁决定什么）

**任何 AI 进项目第一件事**：读这个文件，复述项目状态。

---

## 三类文档的分工

| 类型 | 放哪 | 作用 | 谁读 |
|------|------|------|------|
| **真值文档** | docs/01~15 | 描述每个系统该是什么样 | AI 干活前必读相关文档 |
| **宪法** | AGENTS.md | 规定 AI 的行为准则、范围红线、合作方式 | AI 每次会话必读 |
| **知识库** | 知识库/*.md | 教你（用户）理解概念 | AI 不强制读 |

**关键认知**：知识库是给你看的学习材料，**不是 AI 必读的规矩**。AI 只读 AGENTS.md 和 docs/，不读知识库。如果你想让 AI 也用知识库里的某个概念，**把它写进 AGENTS.md 或 docs/ 才有效**。

## 7 条方案怎么防塌方

```
方案 1：对话开始协议     → 防止新 AI 不读规矩瞎写
方案 2：三项一致原则      → 防止改代码忘改文档
方案 3：数据文件清单      → 防止数字散落各处
方案 4：能力总目录        → 防止加新概念没地基
方案 5：变更影响协议      → 防止新技术推翻旧设计
方案 6：MVP 验收清单      → 防止 MVP 半成品就跑下一步
方案 7：合规巡检          → 定期自查防止规则形同虚设
```

**它们是环环相扣的**：
- 不读宪法（方案1）→ 不知道三项一致（方案2）→ 改代码不改文档 → 拆东墙补西墙
- 不查能力总目录（方案4）→ 加新系统推翻旧地基 → 系统间冲突
- 不走变更协议（方案5）→ 新技术半路插进来 → 原设计推翻重来

## 三项一致原则实操

每次你要做一件事，做完之后**强迫自己回答 3 个问题**：

1. 改了 `docs/` 里相关文档吗？
2. 改了 `data/` 里相关 JSON 吗？
3. 改了 `scripts/` 里相关代码吗？

**该改的都改了，才能 commit**。少改一项都得补上。

提交时在 commit 消息写：
```
[新增] 钓鱼系统，已更新 05-副职业.md 和 data/fishing.json
```

如果只改了代码没改文档：
```
❌ [新增] 钓鱼系统  → 还没同步文档，不能算完成
✅ [新增] 钓鱼系统，已更新 05-副职业.md（数据由 data/fishing.json 提供）
```

## 合规巡检实操

**每周或换会话前**，让 AI 跑一次：

```
请运行合规巡检:
1. 抽查最近 3 个 commit,文档是否同步更新
2. 抽查 data/ 有没有新建未登记的 JSON
3. 抽查 scripts/ 有没有硬编码 if name==...
4. 抽查 AGENTS.md §2.1 MVP 范围有没有越界
5. 抽查能力总目录有没有未登记的新系统

报告每项结果,PASS 或 FAIL+原因
```

AI 跑完告诉你"PASS"或"FAIL 原因 XXX"。FAIL 的话你指不指示补修，由你决定。

**生活类比**：税务局每年来查一次账——不是信不信你的问题，是**定期被强制检查**自然合规。
