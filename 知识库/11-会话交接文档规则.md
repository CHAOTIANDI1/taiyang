# 11 - 会话交接文档规则

## 概念

每次检测到上下文接近满，AI 必须主动写一份"会话交接文档"，确保下次开新会话不丢进度。

生活类比：你下班前要交一份"今天做了啥、明天接着做啥"的交接表给明天的你。

## 功能

让所有项目状态**从一个 AI 会话无损迁移到下一个**。

## 运作方式

### 触发条件

AI 必须在以下情况写交接文档：

| 触发 | 行动 |
|------|------|
| 自检上下文 > 60% | 主动提醒用户，并准备写 |
| 自检上下文 > 80% | 立即停下，写交接，提示换对话 |
| 用户准备关电脑 | 写交接 |
| 用户提交说要换 AI | 写交接 |
| 完成一个里程碑 | 写交接存档 |

### 交接文档放哪

路径：`D:\taiyang\HANDOFF.md`

注意：是项目根目录的 `HANDOFF.md`，每次新交接覆盖旧文件。**不需要保留历史交接**——历史在 Git commit 里。

### 交接文档必含 8 段

```markdown
# 会话交接文档

> 写于：YYYY-MM-DD HH:MM
> 触发：[上下文 80% / 手动 / 里程碑完成]
> AI 模型：opencode-go/glm-5.2

## 1. 会话 ID 与恢复方式
- 本会话 ID：ses_XXXXXXXX
- 恢复：opencode -s ses_XXXXXXXX

## 2. 当前阶段
- [ ] 文档设计阶段（已完成/进行中）
- [ ] MVP 编码阶段
- [ ] MVP 验收阶段
- [ ] 缩小全内容阶段
- [ ] 联机版阶段

## 3. 已完成的任务
- [任务 1] → [结果]
- ...（按时间序）

## 4. 未完成的任务
- [任务 1] 卡在什么阶段 → 下一步该做什么
- [任务 2] 还没开始 → 该不该做、怎么开始

## 5. 当前关键约束
- 当前 MVP 范围红线（AGENTS.md §2.1）
- 当前不能碰的系统
- 当前正在做的事的边界

## 6. 风险点
- 点 1：[风险描述] → 监控方法
- 点 2：...

## 7. 下一步计划
- 下一个具体任务是什么
- 按什么顺序做
- 估计多久

## 8. AGENTS 宪法与文档真源确认
- [x] AGENTS.md 版本：v2.0 当前
- [x] docs/00-总览.md 最新已读
- [x] GitHub 已同步到最新 commit
- [x] 合规巡检结果：[PASS / FAIL]

新会话第一动作：读 AGENTS.md → 读 docs/00-总览.md → 5 句话复述 → 让用户确认 → 继续下一步
```

### 关键原则：先验证再交接

交接前 AI 必须做：
1. `git status` 确认工作区干净
2. `git push` 确认云端已同步
3. 跑合规巡检 5 项（PASS 才能交接干净）
4. 写 HANDOFF.md
5. **提交并推送 HANDOFF.md**

新 AI 进来读 HANDOFF.md → 状态恢复 100%。

## 原理

**外部化记忆** + **显式状态转移**：
- AI 不依赖"上次会话我记得"，依赖"我读了 HANDOFF.md 知道了"
- 状态写在文件里，下次会话从文件加载

类似：游戏存档。你不是每个游戏都从头打，存档读进来等于回到上次。

## 优势

| 优势 | 说明 |
|------|------|
| 零进度丢失 | 8 段全覆盖 |
| 责任分明 | 新 AI 走"对话开始协议"复述后才行动 |
| 跨模型兼容 | 不同 AI 都能读懂（无非是 .md） |
| 项目可追溯 | Git 历史里能看到历次交接 |

## 使用场景

### 场景 1: 我说"我先吃饭，回头继续"

AI：不需要交接（饭点小歇）。你说"我下周再回来" → 写交接。

### 场景 2: AI 自检上下文 75%

AI 主动停下来说："上下文快满了，建议写交接准备换对话。" 你确认 → AI 写 HANDOFF → 你关掉 → 开新对话。

### 场景 3: 里程碑完成（比如 MVP 战斗系统全通过）

AI 写一份交接存档，留作"换 AI 也能继续"的备份。
