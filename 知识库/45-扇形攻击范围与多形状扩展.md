# 45 - 扇形攻击范围与多形状扩展

## 本游戏实例（v2.7 准则）

- **遇到的问题**：
  - **Phase 4.4.6 步骤 3**：需要确定玩家/宠物/红尾蝎的攻击范围形状和数值
  - 改前用 RectangleShape2D 矩形（player hitbox 50×50，pet AttackArea 矩形），无法表达"朝向 + 角度"的扇形攻击
  - 未来 Boss 技能需要圆形 AOE / 直线冲刺 / 十字斩等多种形状，需要可扩展架构
- **专业名词/知识点**：扇形（cone）、半径（radius）、弧度（arc_degree）、对角线、朝向夹角（angle_to）、投影（dot）、归一化（normalized）、双十字（cross）、朝向矩形（rect_dir）
- **技术栈/代码/美术**：GDScript Vector2 数学、`angle_to()`、`dot()`、`normalized()`、`rad_to_deg()`、DamageArea `_is_in_shape` 实现
- **应用过程**：
  1. 用户确认设计准则：32×32 像素方块的对角线 × 1.5 = 攻击半径，扇形弧度 45 度
  2. 数值推导：32 × √2 ≈ 45.25 → × 1.5 ≈ 67.88 ≈ 68 像素（半径）
  3. 在 DamageArea `_is_in_shape` 实现 cone 判定：
     - 距离检查：dist ≤ radius
     - 夹角检查：abs(angle_to) ≤ arc_degree / 2
  4. 一并实现其他 4 种形状：circle / rectangle / rect_dir / cross
  5. 加 `run_self_test()` 验证 5 种形状的边界用例（16 项断言）

---

## 概念

**扇形攻击范围**是"以自身为中心、朝某个方向、限定半径和角度"的攻击区域。**多形状扩展**是把扇形、圆形、矩形、朝向矩形、双十字等多种形状统一在一个工具类里，调用方传 shape_config 切换。

**生活比喻 1（生动形象）**：不同光源照亮的区域形状不同——
- **扇形**：手电筒照出的光锥（朝一个方向、有角度限制）
- **圆形**：灯笼照出的光圈（全方向、有半径限制）
- **矩形**：门框的范围（固定宽高，不旋转）
- **朝向矩形**：激光笔射出的光柱（朝一个方向、长条形）
- **双十字**：风车叶片的十字形（两个互相垂直的长条交叉）

**生活比喻 2（本游戏应用）**：玩家朝右走时按 J 攻击：
- 攻击范围是以玩家为中心、朝右、半径 68 像素、角度 45 度的扇形
- 右前方 30 像素的敌人 → 在扇形内 → 命中
- 右后方 30 像素的敌人 → 不在朝向夹角内 → 不命中
- 正前方 80 像素的敌人 → 超出半径 → 不命中
- 偏角 30 度的敌人 → 超出 45/2=22.5 度 → 不命中

**反例 vs 正例**：
```
❌ 错误（用 RectangleShape2D 矩形，无法表达扇形）：
var rect: RectangleShape2D = RectangleShape2D.new()
rect.size = Vector2(100, 100)  # 只能矩形，不能扇形

✅ 正确（DamageArea 扇形判定）：
var cfg: Dictionary = {"type": "cone", "radius": 68, "arc_degree": 45}
# 距离 ≤ 68 + 朝向夹角 ≤ 22.5 度 → 命中
```

## 功能

- 实现 5 种攻击形状的判定逻辑
- 扇形 cone：朝向 + 半径 + 角度
- 圆形 circle：仅半径（全方向）
- 矩形 rectangle：以自身为中心，宽高（世界坐标，不旋转）
- 朝向矩形 rect_dir：从自身向朝向延伸，长宽
- 双十字 cross：两个互相垂直的矩形交叉
- 数学基础：Vector2 的 length / dot / angle_to / normalized

## 运作方式

### 数值推导（设计真值）

| 项 | 数值 | 推导 |
|----|------|------|
| 像素密度 | 32×32 | AGENTS.md §0 项目身份 |
| 方块对角线 | 32 × √2 ≈ 45.25 | 勾股定理 |
| 攻击半径 | 45.25 × 1.5 ≈ 68 | 对角线 × 1.5（用户确认）|
| 扇形弧度 | 45 度 | 用户确认 |
| 朝向夹角阈值 | 22.5 度 | arc_degree / 2 |

### 扇形 cone 判定（核心代码）

```gdscript
ShapeType.CONE:
    var radius: float = float(config.get("radius", 68.0))
    var arc_deg: float = float(config.get("arc_degree", 45.0))
    if dist > radius:
        return false  # 超出半径
    if dist < 0.01:
        return true  # 自身位置算命中
    var facing_n: Vector2 = facing.normalized() if facing.length() > 0.01 else Vector2(1, 0)
    var angle_to: float = rad_to_deg(facing_n.angle_to(to_target))
    return abs(angle_to) <= arc_deg / 2.0
```

**判定步骤**：
1. 算 `to_target = target_pos - origin`（从攻击者到目标的向量）
2. 算 `dist = to_target.length()`（距离）
3. 距离 > radius → 不在扇形内
4. 算 `angle_to = facing.angle_to(to_target)`（朝向到目标的夹角，弧度）
5. 转度数：`rad_to_deg(angle_to)`
6. `abs(angle_to) ≤ arc_degree / 2` → 在扇形内（朝向左右各 22.5 度）

### 圆形 circle 判定

```gdscript
ShapeType.CIRCLE:
    var radius: float = float(config.get("radius", 68.0))
    return dist <= radius
    # 仅检查距离，不检查朝向（全方向）
```

### 矩形 rectangle 判定

```gdscript
ShapeType.RECTANGLE:
    var width: float = float(config.get("width", 64.0))
    var height: float = float(config.get("height", 64.0))
    var dx: float = abs(to_target.x)
    var dy: float = abs(to_target.y)
    return dx <= width / 2.0 and dy <= height / 2.0
    # 以自身为中心，世界坐标，不旋转
```

### 朝向矩形 rect_dir 判定

```gdscript
ShapeType.RECT_DIR:
    var length: float = float(config.get("length", 68.0))
    var width: float = float(config.get("width", 32.0))
    var facing_n: Vector2 = facing.normalized() if facing.length() > 0.01 else Vector2(1, 0)
    # 投影到朝向方向（正方向才算，向后不算）
    var proj_along: float = to_target.dot(facing_n)
    if proj_along < 0 or proj_along > length:
        return false
    # 投影到垂直方向（绝对值）
    var perp: Vector2 = Vector2(-facing_n.y, facing_n.x)
    var proj_perp: float = abs(to_target.dot(perp))
    return proj_perp <= width / 2.0
```

**关键概念**：
- `dot(facing_n)` 是"目标在朝向方向上的投影长度"——正值表示前方，负值表示后方
- `dot(perp)` 是"目标在垂直方向上的投影长度"——表示左右偏移
- `perp = Vector2(-facing.y, facing.x)` 是 facing 逆时针旋转 90 度的垂直向量

### 双十字 cross 判定

```gdscript
ShapeType.CROSS:
    var length: float = float(config.get("length", 80.0))
    var width: float = float(config.get("width", 24.0))
    var dx: float = abs(to_target.x)
    var dy: float = abs(to_target.y)
    # 水平矩形：长 length 沿 X，宽 width 沿 Y
    var in_horizontal: bool = dx <= length / 2.0 and dy <= width / 2.0
    # 垂直矩形：长 length 沿 Y，宽 width 沿 X
    var in_vertical: bool = dx <= width / 2.0 and dy <= length / 2.0
    return in_horizontal or in_vertical
    # 两个矩形任一命中即可
```

### Vector2 数学基础（核心 API）

| API | 作用 | 例子 |
|-----|------|------|
| `vec.length()` | 向量长度（距离）| `Vector2(3, 4).length() = 5` |
| `vec.normalized()` | 归一化（长度变 1，方向不变）| `Vector2(3, 4).normalized() ≈ (0.6, 0.8)` |
| `a.dot(b)` | 点积（投影长度 × |b|）| `Vector2(1, 0).dot(Vector2(1, 0)) = 1` |
| `a.angle_to(b)` | a 到 b 的夹角（弧度，带方向）| `Vector2(1, 0).angle_to(Vector2(0, 1)) = π/2` |
| `rad_to_deg(rad)` | 弧度转度数 | `rad_to_deg(PI/4) = 45` |

### 朝向夹角判定原理

```
facing = (1, 0)  # 朝右
to_target = (30, 10)  # 目标在右上方

angle_to = facing.angle_to(to_target)
= atan2(cross, dot)
= atan2(facing.x * to_target.y - facing.y * to_target.x, facing.dot(to_target))
= atan2(1*10 - 0*30, 1*30 + 0*10)
= atan2(10, 30)
≈ 0.322 弧度
≈ 18.43 度

abs(18.43) ≤ 22.5（45/2）→ 在扇形内 → 命中
```

## 原理

### 为什么用 32×√2×1.5 推导扇形半径？

- **32**：像素密度（角色/怪物/NPC 统一 32×32 方块）
- **√2**：方块对角线（勾股定理：32² + 32² = 对角线²，对角线 = 32√2 ≈ 45.25）
- **×1.5**：用户确认的"对角线 × 1.5"倍数，让攻击范围比自身稍大
- **结果**：68 像素（约 2.1 身位），玩家能看到攻击范围且不会过大

### 为什么扇形用 angle_to 而不是自己写 atan2？

- `Vector2.angle_to()` 是 Godot 内置函数，已封装 atan2 逻辑
- 带方向（正负）：顺时针为正，逆时针为负
- 返回弧度，需要 `rad_to_deg()` 转度数
- 比自己写 atan2 更简洁，且 Godot 已优化性能

### 为什么朝向矩形用 dot 投影而不是 rotate？

- **dot 投影**：把目标向量投影到朝向方向和垂直方向，得到"前方距离 + 侧偏距离"
- **rotate**：把目标向量旋转 -facing 角度，让朝向变成 X 轴，然后按矩形判定
- dot 投影更直观（直接得到两个方向的距离），且不需要算 facing 的角度
- 性能相近，但 dot 投影代码更清晰

### 双十字为什么用世界坐标不旋转？

- 双十字是两个互相垂直的矩形（水平 + 垂直）
- 世界坐标下水平矩形 = X 方向长 + Y 方向窄，垂直矩形 = X 方向窄 + Y 方向长
- 不旋转的原因：双十字通常代表"对称的十字形"（如风车、神圣十字），不需要朝向
- 如果需要朝向十字（如剑气十字斩），可以扩展为 `cross_dir` 形状

### hit_count 为什么用 -1 表示群体而不是 0？

- 0 会被理解为"不打任何人"（无意义）
- -1 在数学上常表示"无限制"（如数组 slice 的负索引）
- N > 1 表示"最近 N 个"，1 表示"单体最近"
- DamageArea 的判定：`if hit_count == HIT_COUNT_ALL: return in_shape`

## 使用场景

### 场景 1：玩家/宠物/红尾蝎普攻（cone）

```gdscript
var cfg: Dictionary = {"type": "cone", "radius": 68, "arc_degree": 45, "hit_count": 1}
# 朝向 45 度弧、半径 68 像素、最近 1 个敌人
```

### 场景 2：AOE 爆炸（circle）

```gdscript
var cfg: Dictionary = {"type": "circle", "radius": 100, "hit_count": -1}
# 100 像素内所有敌人
```

### 场景 3：陷阱区域（rectangle）

```gdscript
var cfg: Dictionary = {"type": "rectangle", "width": 64, "height": 64, "hit_count": -1}
# 64×64 范围内所有敌人
```

### 场景 4：直线冲刺（rect_dir）

```gdscript
var cfg: Dictionary = {"type": "rect_dir", "length": 200, "width": 40, "hit_count": 1}
# 朝向前方 200 像素、宽 40 像素的最近 1 个敌人
```

### 场景 5：Boss 十字斩（cross）

```gdscript
var cfg: Dictionary = {"type": "cross", "length": 150, "width": 30, "hit_count": -1}
# 十字形范围内所有敌人
```
