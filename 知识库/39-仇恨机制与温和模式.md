# 39 - 仇恨机制与温和模式

## 本游戏实例（v2.7 准则）

- **遇到的问题**：
  - **Phase 4.4.6**：宠物进入战斗模式后，怪物怎么知道该打宠物还是打玩家？如果怪物一直追宠物，玩家就成"打手旁观"了，反之亦然。需要一套"怪物选目标"的规则。同时温和模式宠物不能被攻击（设计：温和模式无敌），但代码层面怎么让怪物"看不见"温和模式宠物？
  - **设计来源**：docs/03 第 50-55 行写"被主角打会立即把仇恨转移到主角上面 5 秒"——这是仇恨机制的来源。但代码怎么实现"5 秒锁定"？
- **专业名词/知识点**：仇恨（Aggro）、仇恨表（Aggro Table）、仇恨锁定（Aggro Lock）、温和模式（Gentle Mode）、无敌（Invincible）、目标选择（Target Selection）、`has_method()` 判断节点类型、可选参数默认值 `attacker: Node = null`
- **技术栈/代码/美术**：GDScript 的 `has_method()` / `get()` 安全访问、可选参数默认值、`_aggro_timer` 计时器、`monster.gd` 的 `_find_nearest_target()` / `take_damage(amount, attacker)` / `_ai_melee_charger()`、`pet.gd` 的 `_is_gentle_mode` / `is_gentle_mode()` / `take_damage()`
- **应用过程**：
  1. 设计"最近目标规则"作为默认：怪物每帧选离自己最近的目标（玩家 or 战斗模式宠物）
  2. 设计"仇恨锁定"作为覆盖：被玩家打 → 锁定玩家 5 秒（`_aggro_timer = 5.0`）
  3. 温和模式实现：宠物 `_is_gentle_mode = true` 时，`take_damage` 直接 return（无敌）+ `_find_nearest_target` 跳过温和模式宠物
  4. 怪物区分玩家和宠物：用 `has_method("is_gentle_mode")` 判断（只有 pet.gd 有此方法，player.gd 没有）
  5. `take_damage` 签名扩展：`(amount)` → `(amount, attacker=null)`，attacker 有默认值向后兼容
  6. 仇恨触发条件：`attacker != null and attacker.name == "Player"` → 锁定玩家（宠物打不触发仇恨）
  7. docs/03 第 98-103 行记录仇恨规则 + 接口扩展

## 概念

**仇恨（Aggro）** = 怪物对某个目标的"憎恨值"，决定怪物优先打谁。MMO 类游戏（魔兽世界/最终幻想 14）的核心机制之一。

**生活比喻 1（生动形象）**：你在路上被两只狗追，一只你刚踢过它一脚，另一只只是路过。你被踢过的那只狗会死死追你，因为你对它有"仇恨"。仇恨值高的目标优先被打。

**生活比喻 2（本游戏应用）**：怪物 `red_scorpion`（红尾蝎）正在追宠物 `pet_001_ham_dog`（火腿狗），玩家从背后砍了它一刀。红尾蝎的"仇恨表"立刻把玩家提到第一位，接下来 5 秒它只会追玩家，不管宠物在它面前怎么晃。5 秒后仇恨过期，回到"谁离我近我打谁"的默认规则。

**温和模式（Gentle Mode）** = 宠物的"和平状态"，不可被攻击也不攻击别人。设计目的是让玩家可以安全带宠物逛街，不担心宠物被打死。

**生活比喻**：温和模式像宠物装了个"隐身罩"——怪物看不见它，它也不会主动咬人。摘掉隐身罩（按 Z 切换战斗模式）后，怪物才能看见它、它才会咬人。

**反例 vs 正例**：
```
❌ 错误（仇恨判定用名字硬编码）：
if attacker.name == "Player":
    _aggro_timer = 5.0
elif attacker.name.begins_with("Pet"):
    pass  # 宠物打不触发仇恨
# 名字改了就崩，扩展性差

✅ 正确（用 attacker.name == "Player" 单一判断）：
if attacker != null and attacker.name == "Player":
    _aggro_timer = AGGRO_DURATION
# 玩家节点名永远是 "Player"（docs/12 规范）
# 宠物节点名是 "Pet"（pet.tscn 根节点名），不触发仇恨
# 未来加 NPC 攻击 → NPC.name != "Player" → 不触发仇恨，符合设计
```

## 功能

- **最近目标规则**：默认怪物打离自己最近的目标
- **仇恨锁定 5 秒**：被玩家打 → 锁定玩家 5 秒，5 秒内不切换目标
- **温和模式无敌**：温和模式宠物 `take_damage` 直接 return
- **温和模式隐身**：怪物 `_find_nearest_target` 跳过温和模式宠物
- **宠物打不触发仇恨**：只有被玩家打才转移仇恨（设计：玩家是主角，宠物是辅助）

## 运作方式

### 仇恨机制代码（monster.gd）

```gdscript
# monster.gd 第 27-29 行：仇恨变量
var _aggro_timer: float = 0.0        # 仇恨计时器（> 0 时强制打玩家）
const AGGRO_DURATION: float = 5.0    # 仇恨锁定时长（docs/03 设计：5 秒）

# monster.gd 第 66-70 行：仇恨倒计时
func _physics_process(delta: float) -> void:
    # ... 其他逻辑 ...
    if _aggro_timer > 0:
        _aggro_timer -= delta
        if _aggro_timer <= 0 and _debug:
            print("[Monster:%s] 仇恨过期，回到最近目标规则" % monster_id)

# monster.gd 第 79-97 行：动态目标选择
func _ai_melee_charger(delta: float) -> void:
    # 仇恨锁定期间不切换目标
    if _aggro_timer <= 0:
        var new_target: Node = _find_nearest_target()
        if new_target != null and new_target != _target:
            _target = new_target
    # ... 攻击逻辑 ...

# monster.gd 第 183-203 行：take_damage 加 attacker 参数
func take_damage(amount: int, attacker: Node = null) -> void:
    if _is_dying:
        return
    current_hp -= amount
    # ... 闪白 + Tween ...
    
    # 4.4.6 仇恨机制：被玩家打时锁定玩家 5 秒
    if attacker != null and attacker.name == "Player":
        _aggro_timer = AGGRO_DURATION
        if _target != attacker:
            _target = attacker
            print("[Monster:%s] 仇恨转移！锁定玩家 5 秒" % monster_id)
    
    if current_hp <= 0:
        start_dying()
```

### 温和模式代码（pet.gd）

```gdscript
# pet.gd 第 21 行：温和模式变量
var _is_gentle_mode: bool = true  # 默认开启温和模式

# pet.gd 第 288-291 行：切换方法
func toggle_gentle_mode() -> void:
    _is_gentle_mode = not _is_gentle_mode
    emit_signal("mode_toggled", _is_gentle_mode)

# pet.gd 第 294-304 行：温和模式无敌
func take_damage(amount: int) -> void:
    if _is_dying:
        return
    if _is_gentle_mode:
        print("[Pet:%s] 温和模式无敌，忽略伤害 %d" % [pet_id, amount])
        return  # 直接 return，不扣血
    current_hp -= amount
    # ... 扣血 + 死亡判定 ...

# pet.gd 第 321-322 行：暴露给怪物的查询方法
func is_gentle_mode() -> bool:
    return _is_gentle_mode
```

### 怪物区分玩家和宠物（monster.gd _find_nearest_target）

```gdscript
# monster.gd 第 134-160 行
func _find_nearest_target() -> Node:
    var player: Node = get_parent().get_node_or_null("Player")
    var best_target: Node = null
    var best_dist: float = 999999.0
    
    # 检查玩家
    if player != null and is_instance_valid(player) and not bool(player.get("_is_dying", false)):
        best_target = player
        best_dist = global_position.distance_to(player.global_position)
    
    # 检查所有战斗模式宠物（遍历场景中的 Pet 节点）
    var pets: Array = get_parent().get_children()
    for child in pets:
        if child == self:
            continue
        # 判断是否是宠物节点（有 is_gentle_mode 方法）
        if child.name != "Player" and child.has_method("take_damage") and child.has_method("is_gentle_mode"):
            # 战斗模式宠物才能被攻击
            if not bool(child.get("_is_gentle_mode", true)) and not bool(child.get("_is_dying", false)):
                var dist: float = global_position.distance_to(child.global_position)
                if dist < best_dist:
                    best_dist = dist
                    best_target = child
    
    return best_target
```

### 代码逐行执行流程

**场景：玩家砍了正在追宠物的红尾蝎一刀**

```
帧 1：玩家按攻击键
  player.gd 的 _check_hit() 用 HitArea 检测到 red_scorpion
  red_scorpion.take_damage(15, self)  # self = Player 节点
    ↓ 进入 monster.gd 的 take_damage
    current_hp -= 15
    闪白 + Tween
    # 检查 attacker.name == "Player" → 触发仇恨
    _aggro_timer = 5.0
    _target = Player  # 锁定玩家
    打印：[Monster:red_scorpion] 仇恨转移！锁定玩家 5 秒

帧 2-N：_physics_process 执行
  _aggro_timer > 0 → 不调用 _find_nearest_target
  _ai_melee_charger 用 _target = Player 攻击玩家

帧 N+5 秒：_aggro_timer 倒计时到 0
  打印：[Monster:red_scorpion] 仇恨过期，回到最近目标规则
  下一帧 _find_nearest_target 重新选最近目标
```

### 关键判断表

| 情境 | 判断 | 行为 |
|------|------|------|
| 玩家攻击怪物 | `attacker.name == "Player"` | 仇恨锁定 5 秒 |
| 宠物攻击怪物 | `attacker.name == "Pet"` | 不触发仇恨 |
| 怪物选目标 - 玩家在场 | `_find_nearest_target` | 选最近目标 |
| 怪物选目标 - 温和模式宠物 | `child.get("_is_gentle_mode", true) == true` | 跳过，不选 |
| 怪物选目标 - 战斗模式宠物 | `child.has_method("is_gentle_mode")` 且 `_is_gentle_mode == false` | 可选为目标 |
| 温和模式宠物被攻击 | `pet.take_damage` 内 `_is_gentle_mode == true` | 直接 return，不扣血 |
| 战斗模式宠物被攻击 | `pet.take_damage` 内 `_is_gentle_mode == false` | 正常扣血 |

## 原理

### 为什么用 `has_method()` 判断节点类型而不是 `is` 关键字？

GDScript 有 `is Node2D` 这种类型判断，但宠物节点是 `CharacterBody2D`，玩家也是 `CharacterBody2D`，无法用类型区分。所以用"是否有 `is_gentle_mode` 方法"来区分——只有 pet.gd 有这个方法，player.gd 没有。

**生活比喻**：你不能用"是不是人"来区分老师和学生（都是人），但可以问"你会讲课吗"——只有老师会。`has_method("is_gentle_mode")` 就是问"你是宠物吗"。

### 为什么 `take_damage` 加 `attacker: Node = null` 默认值？

GDScript 函数参数支持默认值。`attacker = null` 表示"调用时不传 attacker 也合法，默认是 null"。这样：

```gdscript
# 4.4.6 之前：player.gd 调用
body.take_damage(15)                  # ✅ 不传 attacker，attacker = null

# 4.4.6 之后：player.gd 调用
body.take_damage(15, self)            # ✅ 传 self，触发仇恨

# 4.4.6 之后：pet.gd 调用
body.take_damage(10, self)            # ✅ 传 self（宠物），不触发仇恨

# 4.4.6 之后：monster.gd 调用（怪物打玩家/宠物）
target.take_damage(damage)            # ✅ 不传 attacker，不会误触发自己的仇恨
```

**向后兼容**：所有 4.4.6 之前的 `take_damage(amount)` 调用都还能用，不会报错。

### 为什么宠物打不触发仇恨？

设计原因（docs/03 第 103 行）：
- 玩家是主角，应该承担"坦克"职责（挨打）
- 宠物是辅助，应该承担"输出"职责（打人不挨打）
- 如果宠物打也触发仇恨，玩家会失去"控怪"能力，怪物会在玩家和宠物之间反复横跳，体验糟糕

### 仇恨机制的扩展性

未来加 NPC 战斗（如雇佣兵）时：
- 雇佣兵节点名 ≠ "Player" → 攻击怪物不触发仇恨（和宠物一样）
- 怪物 `_find_nearest_target` 加判断 `child.has_method("is_gentle_mode")` 改为 `child.has_method("is_combatant")` → 兼容雇佣兵
- 地基（仇恨 5 秒锁定 + 最近目标规则）不动

## 优势

| 优势 | 说明 |
|------|------|
| 玩家可控 | 玩家可以"拉怪"——砍一刀锁定 5 秒，保护宠物 |
| 宠物安全 | 温和模式无敌，玩家不用担心宠物被打死 |
| 体验自然 | 怪物在玩家和宠物间合理切换，不反复横跳 |
| 接口向后兼容 | `take_damage(amount)` 旧调用不报错 |
| 扩展平滑 | 未来加 NPC 战斗只需加 `has_method` 判断 |
| 调试容易 | `_aggro_timer` 倒计时打印日志，可追踪 |

## 使用场景

### 场景 1：玩家拉怪保护宠物（Phase 4.4.6）

宠物 HP 低 → 玩家砍怪物一刀 → 仇恨转移玩家 5 秒 → 宠物趁机逃跑或回血。

### 场景 2：温和模式逛街（Phase 4.4.6）

玩家带宠物经过怪物区 → 宠物温和模式 → 怪物不攻击宠物 → 玩家也能不战斗直接走过去（如果玩家不动手）。

### 场景 3：战斗模式双线作战（Phase 4.4.6）

玩家按 Z 切战斗模式 → 宠物咬怪物 → 玩家也砍怪物 → 怪物按最近目标规则选一个打 → 玩家砍了怪物一刀 → 怪物锁定玩家 5 秒 → 宠物继续咬。

### 场景 4：未来 NPC 雇佣兵（V1）

雇佣兵节点名 = "Mercenary" → 怪物 `_find_nearest_target` 加判断 → 可选为目标。雇佣兵打怪物不触发仇恨（和宠物一致）。

### 与其他系统对接

- **AttackArea（[[38-攻击范围与DamageArea概念]]）**：AttackArea 命中后调用 `take_damage(amount, attacker)`，attacker 用于仇恨判定
- **状态机（[[33-状态机骨架]]）**：宠物 attack 状态由战斗模式 + 附近怪物触发
- **碰撞层（[[30-碰撞层与碰撞遮罩]]）**：宠物 collision_layer=16，怪物 collision_mask 不检测宠物层，怪物只能通过 `_find_nearest_target` 主动选目标
- **Signal（[[29-Signal信号机制]]）**：`mode_toggled` 信号通知 UI 更新温和模式图标
