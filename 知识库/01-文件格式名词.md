# 01 - 文件格式名词（.md / .json / .gd / .tscn / .gitignore）

## 概念

项目里反复出现的几种文件后缀。全是纯文本，用记事本也能打开。

## 功能

不同后缀分工不同：.md 是给人看的文档；.json 是存数字的数据表；.gd 是给 AI 写的代码；.tscn 是 Godot 的舞台布置图；.gitignore 是"不上传清单"。

## 运作方式

### .md（Markdown 文档）

生活类比：带格式的笔记本。星号 `*` 加粗、井号 `#` 标题、竖线 `|` 表格。

例子：
```
# 这是一级标题
**这是粗体**
- 列表项 1
- 列表项 2
```

怎么打开：记事本能看符号，VS Code/Obsidian 能看排版。

### .json（数据表）

生活类比：一个可以装很多小纸条的抽屉，每张纸条上有"标签 + 内容"。

例子：
```json
{
  "red_scorpion": {
    "name": "红尾蝎",
    "hp": 50,
    "damage": 12
  }
}
```
游戏代码读取这个文件就把"红尾蝎"的血量设成 50。改它 = 改 JSON 里一个数字，不动代码。

### .gd（GDScript 脚本）

生活类比：菜谱——一步步告诉厨师怎么做菜。脚本告诉 Godot 怎么把数据变成游戏行为。

例子：
```gdscript
func _physics_process(delta):
    var direction = Input.get_vector("ui_left", "ui_right", "ui_up", "ui_down")
    velocity = direction * speed
    move_and_slide()
```
这段让角色按方向键移动。

### .tscn（Godot 场景）

生活类比：舞台布置图——摆什么椅子、灯光怎么打。

只能用 Godot 编辑器打开和编辑，不用记事本。

### .gitignore（不上传清单）

生活类比：你搬家时贴的"勿碰物品"清单。

例子（我们项目的内容）：
```
.godot/         ← Godot 自动生成的临时文件
*.tmp          ← 临时文件
data/save_slots/  ← 玩家存档
```
这些不上传到 GitHub。

## 原理

所有这五种文件都是**纯文本文件**——本质就是字节平铺的文字。区别只在于：
- 谁读它（Obsidian/Godot/Git/人类）
- 按什么规则读（Markdown 语法/JSON 键值对/GDScript 语法/Godot 节点树/Git 过滤规则）

## 优势

| 文件 | 优势 |
|------|------|
| .md | 既能人读又能机读，版本管理友好 |
| .json | 数据和代码分离，改数字不动程序 |
| .gd | 像 Python 简单易学，AI 辅助效果好 |
| .tscn | Godot 原生格式，所见即所得 |
| .gitignore | 一行排除一类文件，不用手动删 |

## 使用场景

- 写设计/规矩：用 .md
- 存怪物血量/装备属性：用 .json
- 写游戏行为：用 .gd
- 布置场景（谁是主角怪物放哪）：用 .tscn
- 排除隐私/临时文件：用 .gitignore
