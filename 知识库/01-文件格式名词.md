# 01 - 文件格式名词

> 这些是我们在项目中反复用到的文件类型，用生活类比帮你理解。

---

## .md 文件（Markdown 文档）

**生活类比**：就是带格式的纯文本笔记。像你在手机备忘录里写的日记，但比日记多了排版能力。

| 项 | 说明 |
|----|----|
| 本质 | 纯文本文件 |
| 后缀 | .md |
| 用什么打开 | 任何文本编辑器都可，但 Obsidian/Vscode 能看格式 |
| 能做什么 | 标题、加粗、表格、列表、链接、代码块 |
| 我们用来 | 存所有设计文档（docs/*.md 和 AGENTS.md） |

**关键认知**：.md 文件你电脑上右键用记事本打开也行，只是看起来全是符号没排版。装了 Obsidian 或 VS Code 就有漂亮的排版显示。

---

## .json 文件（数据表）

**生活类比**：一张可以装很多小纸条的抽屉。上面写"配菜表"，抽屉里每张纸条写着"鱼香肉丝的配方"。你换菜的话只换抽屉里的那张纸条，不用换抽屉本身。

| 项 | 说明 |
|----|----|
| 本质 | 纯文本文件，用花括号 `{}` 包着键值对 |
| 后缀 | .json |
| 长什么样 | `{"name": "红尾蝎", "hp": 50, "damage": 12}` |
| 我们用来 | 存所有"可调的数字"——怪物血量/物品属性/食谱等 |
| 怎么改 | 用记事本/Obsidian/VS Code 打开，改数字，保存即可 |

**对比**：你以前玩的游戏可能把怪物血量写死在代码里。改一下要程序员重新编译游戏。我们设计的规矩是：**血量、攻击力这些数字全部躺在 JSON 里**，改 = 改文本文件一个数字，永远不用动代码。这就是 AGENTS.md §1.2 说的"数据表驱动"。

**JSON 长什么样**（举例 `monsters.json`）：
```json
{
  "red_scorpion": {
    "name": "红尾蝎",
    "hp": 50,
    "damage": 12,
    "speed": 80,
    "ai_template": "melee_charger"
  }
}
```
游戏代码读取这个文件，"红尾蝎"就有 50 血、12 攻击。你要让它变强 = 改 50 成 100，保存，游戏自动生效。

---

## .gd 文件（GDScript 脚本）

**生活类比**：冰箱里贴的菜谱——一步步告诉厨师怎么把食材变成菜。GDScript 脚本就是告诉 Godot 怎么把数据变成画面的菜谱。

| 项 | 说明 |
|----|----|
| 本质 | 纯文本文件，写 GDScript 代码 |
| 后缀 | .gd |
| 用什么打开 | Godot 编辑器内置编辑器，或 VS Code |
| 我们用来 | 写所有游戏行为代码 |
| 跑起来要 | Godot 引擎执行 |

**对比**：.json 是"食材清单"（红尾蝎 50 血），.gd 是"做菜步骤"（怎么读 50 血怎么扣 50 血怎么显示在屏幕上）。两件事分开，数据改起来灵活，代码不变。

---

## .tscn 文件（Godot 场景）

**生活类比**：一张舞台布置图——这个台上摆几个椅子、灯光怎么打、谁先出场。Godot 场景就是游戏里一个"舞台"的布置图。

| 项 | 说明 |
|----|----|
| 本质 | 纯文本文件，Godot 自己的格式 |
| 后缀 | .tscn |
| 用什么打开 | Godot 编辑器 |
| 我们用来 | 定义每个地图/角色/UI 长什么样 |
| 跟 .gd 的关系 | .tscn 描述"摆什么"，.gd 描述"动什么" |

---

## .gitignore 文件

**生活类比**：搬家公司来之前你在门口贴的"勿碰物品"清单——这几箱私人物品不带走。

| 项 | 说明 |
|----|----|
| 本质 | 纯文本文件 |
| 作用 | 告诉 Git "这些文件不要上传到 GitHub" |
| 我们用来 | 排除 `.godot/`（引擎自动生成的临时文件）、密码、本地配置等 |

---

## 一句话总结

```
.md   = 你读的笔记 / 设计文档 / 规矩
.json = 你修改的数值 / 怪物血量 / 食谱配方
.gd   = AI 写的代码 / 游戏行为
.tscn = 游戏舞台布置图
.gitignore = 不上传 GitHub 的私人物品清单
```

所有这些都是**纯文本文件**，你能用记事本打开任何一个看。区别只是用哪个工具看最方便。
