# 02 - 技术栈与工具全景

## 概念

做这个游戏你会用到的所有工具，以及它们的关系。

## 功能

每个工具有自己的职责，分工协作完成"设计 → 代码 → 测试 → 备份 → 上架"全流程。

## 原理

工具之间靠**文件**协作，不靠互相调用：
- OpenCode 生成 .gd / .json / .md 文件
- Godot 读 .gd 和 .json 跑游戏
- Git 监视文件夹，把变化提交存档
- GitHub 是 Git 存档的云端备份

## 优势

- 每个工具只做自己最擅长的事，**AI 写代码比人快，Godot 引擎比自研省 100 倍工作量**
- 所有产物是文本文件，**可追溯、可备份、可恢复**
- 工具之间不绑定，**换 AI/换引擎都还能用现有文档和数据**

## 使用场景

### 场景 1：写代码
你用 OpenCode 说"做角色移动" → 我生成 .gd 文件 → 你粘贴到 Godot → 运行。报错 → 你贴给我 → 我改。

### 场景 2：改数值
你打开 `data/monsters.json` → 用 VS Code/记事本 → 改红尾蝎血量 50→100 → 保存 → 重启游戏 → 自动生效。**不用动一行代码**。

### 场景 3：保存工作
你做完一段功能 → 打开 GitHub Desktop → 写描述 → 点 Commit → 点 Push → 云端同步。明天电脑坏了也不丢。

### 场景 4：换电脑继续
你换一台电脑 → 装 Git → `git clone https://github.com/CHAOTIANDI1/taiyang` → 全部代码/文档下来了 → 继续干。
