## 每个工具的作用

### 1. Godot 4.x（游戏引擎）

**是什么**：免费的游戏开发软件，类似"游戏工厂"。

| 做什么 | 类比 |
|--------|------|
| 创建场景（角色、地图、UI） | 布置舞台 |
| 写 GDScript 脚本 | 写演员台词和动作 |
| 调节点参数 | 调灯光/位置/大小 |
| 导出 .exe | 把整场演出录成 DVD |

你现在装的就是这个。它是你 90% 时间会用的工具。

### 2. GDScript（编程语言）

**是什么**：Godot 自己的编程语言，长得像 Python。

**你不用从头学**：AI（我）会给你写好每一段代码，你只要复制粘贴到 Godot 里就行。学着学着你会看懂关键的部分。

**生活类比**：你不一定要会做菜，但你要会"尝味道判断咸淡"。GDScript 你不用会写，但要能看出来"这段是控制移动的、那段是控制攻击的"。

### 3. OpenCode（我·AI）

**是什么**：你正用的这个终端工具。

**怎么用**：
- 用自然语言告诉我"我要做角色移动"
- 我给你 GDScript 代码 + 告诉你在 Godot 里点哪里
- 你照做后有问题，把报错贴给我，我修

**重点**：每次新对话开始时，要让我先读 `AGENTS.md`。这是 7 条方案之一。

### 4. Git + GitHub Desktop

**是什么**：超级存档系统 + 云端备份。

**生活类比**：
- **Git**：游戏存档功能。每写完一段代码 = 手动存一次，可以回退到任何一次存档点。
- **GitHub**：云端存档。你电脑坏了代码也不丢，全世界都能看到（如果你开源）。

| 概念 | 说明 |
|------|------|
| Commit | 给当前工作打个"存档点"，写一句话描述改了啥 |
| Push | 把存档点同步到 GitHub 云端 |
| Pull | 从云端拉回最新版（你换电脑时） |
| 仓库 | 一个项目的所有存档点合起来 |

**操作就三步**：写完代码 → 打开 GitHub Desktop → 写描述 → 点 Commit → 点 Push。

### 5. Obsidian（个人知识库）

**是什么**：你刚下载的笔记软件，专门读 .md 文件。

**怎么用**：
- 把 `D:\taiyang\知识库\` 这个文件夹拖进 Obsidian 当"仓库"
- 以后所有"我新学到的概念"都写在这里
- 可以搜索、双链、标签管理

**生活类比**：你的私人图书馆。这本书（这个 .md）讲"JSON 是什么"，那本书讲"Git 怎么用"，找东西一搜就到。

### 6. Aseprite（未来美术工具）

**是什么**：像素画 + 动画制作工具，约 ¥60。

**什么时候用**：MVP 代码跑通后做美术时才用。现在不用装也行。

### 7. Gemini（未来生图工具）

**是什么**：谷歌的 AI，能生成图片。

**什么时候用**：做美术时让 AI 生成角色概念图，然后用 Aseprite 像素化。

## 工具之间的协作关系

```
你 → 用 OpenCode（我）→ 生成 .gd 代码 → 粘贴到 Godot
                       → 生成 .json 数据 → 放进项目
                       → 生成 .md 文档 → 放进 docs/

Godot → 运行游戏 → 测试手感
     → 出错 → 报错贴给我 → 我改

做完一段 → GitHub Desktop 提交 → 存到 GitHub 云端
```

**每周做一次**：用 GitHub Desktop 检查 "有没有改代码没改文档" ——这就是 7 条方案中的"自我审查"。
