# 13 - 音效与音乐资源处理

## 概念

游戏里两种声音 —— **BGM**（背景音乐，持续播放）和**音效**（一次性触发，比如命中"叮"声）。

## 功能

- BGM：营造氛围，让玩家有情绪
- 音效：给动作反馈（命中、走路、升级），让操作有"手咚"感

**打击感的一半来自音效**——没有命中"叮"一声，再好的画面也觉得软。

## 运作方式

### 资源格式

| 类型 | 格式 | 大小 |
|------|------|------|
| BGM | .ogg（Ogg Vorbis） | 1~3 MB/曲 |
| 音效 | .wav（短促效果） | <100 KB/个 |

Godot 原生支持这两种格式，**不用装插件**。

### 在 Godot 里怎么用

**BGM 节点**：`AudioStreamPlayer`
- 一首曲子循环播放
- 切换地图时切 BGM（可以做淡入淡出）

**音效节点**：`AudioStreamPlayer2D`（2D 定位）或 `AudioStreamPlayer`（全屏通用）
- 角色挥剑 → 播 wav 文件 0.3 秒
- 怪物被打 → 播"叮"声 0.1 秒

### 代码层

```gdscript
# 打击音效
func play_hit_sound():
    $HitSound.play()    # AudioStreamPlayer 节点自动播放
```

## 原理

声音在游戏中是**事件触发**的——某个事件发生（攻击命中）→ 代码调音频节点播放。
BGM 是**常态播放**——一进地图就开始，地图切走才停。

## 优势

| 好的音效 | 差的音效 |
|---------|---------|
| 命中"咚"声 → 玩家觉得打到了 | 没声音 → 砍了像砍空气 |
| 升级"叮咚" → 有满足感 | 没声音 → 升级没仪式 |
| 走路脚步声 → 节奏感 | 没声音 → 走得像飘 |

### 音效配置文件（项目里怎么登记）

我们项目所有音效会登记在 `data/sounds.json`（**新加一项到 §1.4 清单**）：

```json
{
  "bgm_village":      { "file": "bgm/village.ogg", "loop": true },
  "sfx_hit_sword":    { "file": "sfx/hit_sword.wav", "loop": false },
  "sfx_levelup":      { "file": "sfx/levelup.wav",  "loop": false },
  "sfx_fish_splash":  { "file": "sfx/splash.wav",   "loop": false }
}
```

改音效 = 改 JSON 一个文件路径，不改代码。和怪物数据走一样的"数据驱动"思路。

## 使用场景

### 场景 1：做战斗反馈

写攻击代码时，加：
```gdscript
func on_hit():
    play_hit_sound()      # 音效
    spawn_particles()     # 粒子（见 15 笔记）
    shake_camera()         # 屏震
    spawn_damage_number() # 飘字
```
4 件事叠起来就是饥荒式打击感——**音效在最关键位置**。

### 场景 2：换地图自动 BGM

```gdscript
# 进入新手村时
AudioManager.play_bgm("bgm_village")

# 进入野外 A 时
AudioManager.crossfade_to("bgm_field")
```

新加一个 AudioManager 单例统一管 BGM/音效播放。**加入能力总目录和 JSON 清单**。

### 场景 3：钓鱼的小水花

钓鱼成功 → 播 `sfx_fish_splash` → 配合粒子效果（水面溅起 dealt）。

## MVP 范围

| 项 | MVP | 后续 |
|----|:--:|:--:|
| 场景 BGM（5 张地图各 1 曲） | ⏳ 可以后做 | ✅ |
| 战斗音效（挥砍/命中/受击） | ✅ 应做 | ✅ |
| 升级音效 | ✅ 应做 | ✅ |
| 走路脚步声 | ⏳ 可后做 | ✅ |
| UI 点击音效 | ⏳ | ✅ |

**MVP 至少要做战斗音效 + 升级音效**——这是手感验证不可缺的一环。
