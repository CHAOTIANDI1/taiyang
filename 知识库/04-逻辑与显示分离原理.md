# 04 - 逻辑与显示分离原理

## 概念

"游戏逻辑"（算账）和"画面显示"（看戏）必须分开写。逻辑算出结果，告诉画面"你该表演什么"，画面不直接知道账本内容。

## 功能

将来联机版把"算账的逻辑"搬到服务器，"看戏的画面"留在玩家电脑——代码不用重写。

## 运作方式（举两个反例+正例）

### 反例 1：逻辑直接改画面

```gdscript
# attack_manager.gd（错误）
func attack(enemy):
    enemy.hp -= 10                 # 扣血（逻辑）
    enemy.hp_bar.value = enemy.hp  # 直接改血条 UI（画面）
    enemy.play_animation("hit")    # 直接播动画（画面）
```

问题：联机版时服务器要算扣血，但服务器不该管玩家电脑上的血条怎么画。

### 反例 2：画面直接改逻辑

```gdscript
# enemy.gd（错误）
func _on_hit_animation_finished():
    hp += 5      # 动画播完就加血？这是"画面驱动逻辑"，会失控
```

问题：画面出错时（动画没播完）逻辑数据会混乱。

### 正例：逻辑广播事件，画面自己反应

```gdscript
# attack_manager.gd（逻辑层）
func attack(enemy):
    var damage = calculate_damage()      # 算伤害
    enemy.take_damage(damage)            # 调用敌人扣血函数
    enemy.emit_signal("damaged", damage) # 广播事件"被打到了"
```

```gdscript
# enemy.gd（接收层，跟逻辑挂钩但画面另算）
func take_damage(amount):
    hp -= amount
    if hp <= 0: emit_signal("died")

# enemy_view.gd（画面层，自己听信号自己决定表演）
func _on_damaged(amount):
    play_animation("hit")
    spawn_damage_number(amount)
    flash_white()
```

逻辑只算"掉了 12 血"，画面听到"被打到了"就自己播"闪白 + 数字 + 动画"。**两边分工清楚**。

## 原理

逻辑和画面是**两种不同性质的代码**：
- 逻辑：**纯计算无副作用**——同样输入同一个敌人，掉的血永远一样。可搬到任何机器跑，包括服务器。
- 画面：**有副作用，依赖屏幕**——播动画需要 GPU、需要时间、可能卡顿。

混在一起 = 服务器没办法只跑逻辑。分开 = 服务器跑逻辑，客户端跑画面，**两者通过网络传"事件"同步**。

## 优势

| 优势 | 说明 |
|------|------|
| 单机到联机不用重写 | 服务器跑逻辑，客户端跑画面，已经天然分开 |
| 单元测试容易 | 算法纯算账独立可测，不依赖 Godot 渲染 |
| 调试容易 | 调试逻辑只看输入输出，不管动画面面 |
| 性能优化容易 | 逻辑可以批量处理，画面按帧渲染 |

## 使用场景

### 场景 1：你在做攻击

逻辑层（`attack_manager.gd`）：算伤害 = 武器伤害 + 力量 - 敌人防御。
画面层（UI）：听到"被攻击"信号 → 飞出伤害数字 → 播闪白 → 摄像机震动。

### 场景 2：做存档

逻辑层：把角色数据存进 SaveManager.save()。
画面层：SAVED 字样淡入淡出。

### 场景 3：将来做联机

服务器端（搬过去）：计算伤害 + 通知敌人死亡 + 处理掉落
客户端（留下）：播动画 + 飞数字 + 显示掉落物图标
**两边通过网络传"伤害事件"同步**
