# 34 - 场景实例化与子场景（Scene Instancing）

## 本游戏实例（v2.7 准则）

- **遇到的问题**：Phase 4.4.2 宠物场景 `scenes/pet/pet.tscn` 创建完后，需要在主世界 `scenes/player/world.tscn` 里加一只测试宠物（火腿狗）。如果直接在 world.tscn 里复制粘贴 Pet 节点结构（CharacterBody2D + CollisionShape2D + ColorRect），将来改宠物结构（比如把 ColorRect 换成 Sprite2D）就要同时改 N 处。需要一种"定义一次，复用多次"的方式。
- **专业名词/知识点**：场景（Scene）、实例化（Instancing）、子场景（Child Scene）、PackedScene、ExtResource、节点覆盖（Node Override）
- **技术栈/代码/美术**：Godot 4 的 `.tscn` 文件格式、`[ext_resource type="PackedScene"]`、`[node name="X" parent="." instance=ExtResource("id")]`、world.tscn 第 5 行和第 89-91 行
- **应用过程**：
  1. 先把宠物结构定义在 `scenes/pet/pet.tscn`（根节点 Pet + 子节点 PetShape + ColorRect）
  2. 在 world.tscn 顶部 `[ext_resource]` 段加 `path="res://scenes/pet/pet.tscn" id="3_pet"` 引用
  3. 在 world.tscn 末尾用 `[node name="TestPet" parent="." instance=ExtResource("3_pet")]` 实例化
  4. 在实例化节点下覆盖两个属性：`position = Vector2(350, 400)` 和 `pet_id = "pet_001_ham_dog"`
  5. 改 pet.tscn 结构时，所有实例（TestPet 等）自动跟随，不用逐个改

---

## 概念

**场景实例化** = 把一个 `.tscn` 文件作为"模板"，在另一个 `.tscn` 文件里"复制一份"出来，可以覆盖部分属性。模板改了，所有实例自动跟随。

**PackedScene** = Godot 对"场景文件"的内部表示。一个 `.tscn` 文件加载到内存就是一个 PackedScene 对象，可以多次实例化。

**生活比喻 1（生动形象）**：蛋糕模具。你做一个"星形模具"（pet.tscn），用这个模具可以烤 N 个星形蛋糕（实例）。每个蛋糕可以撒不同糖粉（覆盖属性：颜色、口味），但形状都来自同一个模具。改模具的形状，下次烤出来的蛋糕自动是新形状。

**生活比喻 2（本游戏应用）**：
- `pet.tscn` 是"宠物模具"——定义了 Pet 节点 + CollisionShape2D + ColorRect 的结构
- `world.tscn` 用 `instance=ExtResource("3_pet")` 烤出一个 TestPet 蛋糕
- 在 TestPet 上覆盖 `pet_id = "pet_001_ham_dog"` 和 `position = Vector2(350, 400)`——撒"火腿狗"和"位置 350,400"的糖粉
- 未来加铁拳宠物：再烤一个蛋糕，撒 `pet_id = "pet_002_iron_fist"` 的糖粉
- 改模具（pet.tscn 把 ColorRect 换成 Sprite2D）→ 所有宠物蛋糕自动换成 Sprite2D

**反例 vs 正例**：
```
❌ 错误（复制粘贴节点）：
world.tscn 里直接写：
[node name="TestPet" type="CharacterBody2D"]
collision_layer = 16
...（重复 pet.tscn 全部内容）
[node name="PetShape" type="CollisionShape2D" parent="TestPet"]
...
[node name="ColorRect" type="ColorRect" parent="TestPet"]
...
→ 改 pet.tscn 结构，world.tscn 不跟随，要手动改 N 处

✅ 正确（实例化）：
world.tscn 里只写：
[ext_resource type="PackedScene" path="res://scenes/pet/pet.tscn" id="3_pet"]
...
[node name="TestPet" parent="." instance=ExtResource("3_pet")]
position = Vector2(350, 400)
pet_id = "pet_001_ham_dog"
→ 改 pet.tscn 结构，所有实例自动跟随
```

---

## 功能

- 把一个场景作为模板复用到多个地方
- 实例可覆盖模板的部分属性（position、变量值等）
- 模板修改时所有实例自动跟随，不用逐个改
- 节点结构、脚本、子资源全部跟随模板

## 运作方式

### world.tscn 实例化 pet.tscn 的代码（第 1-5 行 + 89-91 行）

```ini
[gd_scene load_steps=10 format=3 uid="uid://b1world01taiyang"]

[ext_resource type="Script" path="res://scripts/player/player.gd" id="1_player"]
[ext_resource type="Script" path="res://scripts/enemy/monster.gd" id="2_monster"]
[ext_resource type="PackedScene" path="res://scenes/pet/pet.tscn" id="3_pet"]
;                                                  ↑ 关键：type="PackedScene" 表示这是场景资源

...（中间是其他节点定义）

[node name="TestPet" parent="." instance=ExtResource("3_pet")]
;                  ↑ 实例化节点的父节点是根节点"."（World）
;                                    ↑ instance= 表示这个节点是从外部场景实例化来的
position = Vector2(350, 400)
;       ↑ 覆盖属性：把宠物放在 (350, 400) 位置
pet_id = "pet_001_ham_dog"
;      ↑ 覆盖属性：设置 pet.gd 的 @export 变量 pet_id 为火腿狗
```

### 代码逐行执行流程

**场景：玩家按 F5 启动游戏，加载 world.tscn**

```
1. Godot 读 world.tscn 文件
2. 遇到 [ext_resource type="PackedScene" path="res://scenes/pet/pet.tscn" id="3_pet"]
   → 把 pet.tscn 加载到内存作为 PackedScene 资源，起个别名 id="3_pet"
3. 遇到 [node name="TestPet" parent="." instance=ExtResource("3_pet")]
   → 调用 PackedScene.instantiate() 创建一份 pet.tscn 的副本
   → 副本的根节点命名为 "TestPet"，挂到 World 节点下
4. 处理 TestPet 下面的覆盖属性：
   position = Vector2(350, 400) → 把 TestPet 的位置设为 (350, 400)
   pet_id = "pet_001_ham_dog"   → 把 TestPet 的 pet_id 变量设为 "pet_001_ham_dog"
5. World 节点的子节点树最终是：
   World
   ├── BackgroundColor
   ├── Ground
   ├── Player
   ├── TrainingDummy
   ├── RedScorpion
   └── TestPet（从 pet.tscn 实例化，位置 350,400，pet_id=火腿狗）
       ├── PetShape（从 pet.tscn 继承）
       └── ColorRect（从 pet.tscn 继承，颜色由 pet.gd 启动时按 pet_id 读 pets.json 覆盖）
```

### 在 Godot 编辑器里怎么看实例化节点

1. 打开 world.tscn
2. 在场景树里看 TestPet 节点——它会有一个"小图标"表示是实例化来的（不是普通节点）
3. 展开 TestPet，里面的 PetShape 和 ColorRect 是"灰色"的——表示继承自模板，不能直接改
4. 要改 PetShape 或 ColorRect，必须改 pet.tscn，所有实例自动跟随

## 原理

Godot 的场景系统是**组合模式**（Composite Pattern）的实现：
- 一个场景 = 一棵节点树 + 资源引用 + 节点属性
- 场景可以嵌套——一个场景的节点可以是另一个场景的实例
- 实例化时 Godot 创建模板节点的"深拷贝"，但共享资源（脚本、形状等）

**PackedScene.instantiate() 的本质**：
- 按模板的节点树结构创建新节点
- 复制所有节点的属性到新节点
- 共享子资源（RectangleShape2D 等被多个实例引用时不重复创建）

**节点覆盖的规则**：
- 只能覆盖实例化节点的"自己定义的属性"（position、script 变量等）
- 不能覆盖子节点的属性（要让 TestPet 的 ColorRect 颜色不同，得在 pet.gd 启动时按 pet_id 读 pets.json 设置，不在 world.tscn 覆盖）

**为什么用实例化而不是代码 `load().instantiate()`？**
- world.tscn 里的实例化是"编辑器时间"的——Godot 启动时自动加载
- 代码实例化是"运行时间"的——适合蛋孵化时动态生成宠物
- 4.4.2 测试用编辑器时间实例化（简单直接）
- 4.4.3 蛋孵化用代码 `pet_scene.instantiate()` 动态生成

---

## 优势

| 优势 | 说明 |
|------|------|
| 模板一次定义，多处复用 | pet.tscn 改一次，所有宠物实例自动跟随 |
| 实例属性可覆盖 | 同一模具烤不同口味蛋糕（覆盖 pet_id） |
| 节点结构自动同步 | 加 PetShape 子节点不用改 world.tscn |
| 编辑器可视化 | 场景树里能展开 TestPet 看到子节点结构 |
| 升级平滑 | 从"编辑器实例化"升级到"代码实例化"，模板（pet.tscn）不动 |

## 使用场景

### 场景 1：MVP 测试宠物（Phase 4.4.2 当前实现）

`world.tscn` 用 `instance=ExtResource("3_pet")` 加一只测试宠物火腿狗，验证 pet.gd 加载、温和模式切换等功能。

### 场景 2：蛋孵化动态生成（Phase 4.4.3 计划）

蛋孵化时玩家可能拿到不同宠物，不能在 world.tscn 写死。改用代码：
```gdscript
var pet_scene: PackedScene = load("res://scenes/pet/pet.tscn")
var pet: CharacterBody2D = pet_scene.instantiate()
pet.pet_id = "pet_001_ham_dog"  # 根据蛋类型决定
pet.position = player.position + Vector2(-50, 0)  # 出现在玩家身边
get_parent().add_child(pet)
```

### 场景 3：怪物刷新（Phase 4.5 计划）

未来怪物也用实例化——`monster.tscn` 作为模板，刷新点用代码 `instantiate()` 动态生成不同 monster_id 的怪物。

### 与其他系统对接

- **DataManager**：实例化后 `pet._ready()` 自动调用 `DataManager.get_pet(pet_id)` 加载数据
- **SaveManager**：存档时保存 pet_id 和 position，读档时实例化对应宠物
- **InputManager**：实例化的宠物自动响应 `pet_toggle_mode` 输入
